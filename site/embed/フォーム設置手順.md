# かんたんWEB審査フォーム WordPress設置手順

フォーム本体（`shinsa.html`）は Netlify に置き、WordPressからは **iframe で呼び出すだけ**。
テーマのCSSと干渉せず、ID重複などの事故も起きません。

```
　［Netlify］ https://regal-babka-f817bb.netlify.app/shinsa.html  ← フォーム本体（1ファイル）
　　　▲
　　　│ iframe で呼び出し（高さは自動調整）
　　　│
　［WordPress］
　　├─ A. 審査ページ /shinsa-2 …… ページ直接表示（開いたらフォームが出る）
　　└─ B. トップ・共通CTA …………… ボタン→ポップアップ（押すと重なって開く）
```

> フォームを直したいときは Netlify の `shinsa.html` を差し替えるだけ。
> WordPress側のコードは触らなくてOK（同じURLを見に行くため自動で最新になる）。

---

## A. 審査ページ（/shinsa-2）に「直接表示」で置く

**開いたらそのままフォームが表示される**置き方です。審査専用ページ向け。

### 手順
1. WordうPress管理画面 → 固定ページ →「/shinsa-2」を編集
2. 「＋」→「カスタムHTML」ブロックを追加
3. `shinsa-embed.html` の中身を丸ごと貼り付け（下記）
4. 公開

### 貼り付けるコード（shinsa-embed.html）
```html
<div id="carmel-shinsa-embed" style="max-width:960px;margin:0 auto;">
  <iframe
    id="carmel-shinsa-frame"
    data-src="https://regal-babka-f817bb.netlify.app/shinsa.html"
    title="かんたんWEB審査フォーム"
    loading="lazy"
    scrolling="no"
    style="width:100%;border:0;display:block;height:1400px;overflow:hidden;background:#f1f5f9;"
  ></iframe>
</div>
<script>
(function(){
  var frame=document.getElementById('carmel-shinsa-frame');
  if(!frame) return;
  if(frame.getAttribute('data-src')) frame.src=frame.getAttribute('data-src');
  window.addEventListener('message',function(e){
    var d=e.data;
    if(d && typeof d.carmelShinsaHeight==='number' && d.carmelShinsaHeight>200){
      frame.style.height=d.carmelShinsaHeight+'px';
    }
  });
})();
</script>
```

## B. トップページ・サイト共通CTAに「ボタン→ポップアップ」で置く

**ボタンだけ置き、押すとフォームが画面の上に重なって開く**置き方です。
トップページやLP、フッター共通など、いろんなページから同じフォームを呼び出せます。

### 手順
1. ボタンを置きたいページ（トップ等）を編集、または共通ウィジェット/フッターへ
2. 「＋」→「カスタムHTML」ブロックを追加
3. `shinsa-popup-embed.html` の中身を丸ごと貼り付け
4. 必要ならボタン文言（かんたんWEB審査を始める）を変更
5. 公開

> コードは `embed/shinsa-popup-embed.html` にそのまま入っています（長いのでファイル参照）。
> スタイルは全て `#carmel-shinsa-pop` 配下にスコープ済みで、テーマと干渉しません。

---

## URLについて（重要）

- `data-src` の `https://regal-babka-f817bb.netlify.app/shinsa.html` が
  **フォーム本体の場所**です。
- 独自ドメイン（例：carmelonline.jp配下）に載せ替えたら、
  A・B両方の `data-src` をそのURLに書き換えてください。

## 動作の確認ポイント
- スマホ・PC両方で開いてみる（ポップアップはスマホで全画面表示）
- ステップを進めて「入力してください」の赤字が出るか
- 実際に1件テスト送信 → スプレッドシート／Drive／Slack／Asana／サンクスメール到達を確認
