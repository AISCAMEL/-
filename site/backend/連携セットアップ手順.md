# 審査フォーム 4媒体連携 セットアップ手順

申込フォーム（`shinsa.html`）を送信すると、**1回で以下4か所に自動で流れる**ようにします。

```
　お客様がフォーム送信
　　        │
　　        ▼
　　【GAS 中継ハブ】gas-shinsa-hub.gs（1本の窓口）
　　  ├─① Google スプレッドシート（申込台帳に1行追加）
　　  ├─② Slack #ローン審査 へ通知
　　  ├─③ Asana「カーメルFC」ボードにタスク作成
　　  ├─④ WordPress（任意・既存の受付へ転送）
　　  └─   免許証などの画像 → Google Drive に保存し各所にリンク添付
```

> **慣れるまでは全部オン**にして併走 → 不要になった媒体は、スクリプト冒頭の
> `SEND_TO` を `false` にするだけで、1行で止められます。

---

## 準備するもの（4つのID・URL）

| # | 何が必要か | どこで取るか |
|---|---|---|
| 1 | **スプレッドシートID** | 台帳用シートを新規作成 → URLの `/d/●●●/edit` の `●●●` |
| 2 | **Drive フォルダID** | 添付保存用フォルダを新規作成 → URL末尾の文字列 |
| 3 | **Slack Incoming Webhook URL** | 下記「Slack設定」参照（#ローン審査 用） |
| 4 | **Asana 個人アクセストークン** | 下記「Asana設定」参照 |

Asana のボード・ワークスペースIDは**設定済み**です（カーメルFC）。

---

## STEP 1：GAS プロジェクトを作る

1. [script.google.com](https://script.google.com) を開き **「新しいプロジェクト」**
2. 中身を全消しして、`gas-shinsa-hub.gs` の内容を**丸ごと貼り付け**
3. 冒頭の `CONFIG` を自分の値に書き換える（上の4つ）

## STEP 2：Slack の Webhook を作る（②の通知用）

1. https://api.slack.com/apps → **Create New App**（From scratch）→ ワークスペース選択
2. 左メニュー **Incoming Webhooks** をオン
3. **Add New Webhook to Workspace** → 投稿先に **#ローン審査** を選択 → 許可
4. 発行された `https://hooks.slack.com/services/...` を CONFIG の `SLACK_WEBHOOK_URL` に貼る

## STEP 3：Asana のトークンを作る（③のタスク作成用）

1. Asana → 右上アイコン → **設定** → **アプリ** → **開発者コンソール**
   （または https://app.asana.com/0/my-apps ）
2. **Personal access tokens** → **Create new token** → 名前をつけて発行
3. 表示されたトークンを CONFIG の `ASANA_TOKEN` に貼る（**一度しか表示されません**）

> 送信先ボードは「カーメルFC」に設定済み。別のボードにしたい場合は、
> `ASANA_PROJECT_GID` を変更してください。
> （本店=1212957542191186／山梨=1213077965542217／千葉=1213077965542227／
> 　小田原=1213077965542236／カーメルFC=1213728225406661）

## STEP 4：デプロイ（フォームからのURLを発行）

1. GASエディタ右上 **デプロイ** → **新しいデプロイ**
2. 種類（歯車）→ **ウェブアプリ**
3. 設定：
   - 説明：`審査フォーム中継ハブ`
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
4. **デプロイ** → 初回は権限承認（自分のGoogleアカウントで許可）
5. 発行された **ウェブアプリURL（`…/exec`）** をコピー

## STEP 5：フォームに接続

`shinsa.html` の上部にある設定を書き換えます：

```html
<script>
  window.SHINSA_ENDPOINT = "ここにSTEP4の /exec URL を貼る";
</script>
```

これで完了。フォームを送信すると4か所に届きます。

## 動作テスト

- **GAS単体テスト：** エディタで関数 `TEST_run` を実行 → シート追加・Slack通知・Asanaタスクができるか確認（ダミーデータ）
- **フォーム通し：** 実際に `shinsa.html` から1件送信し、4か所に届くか確認
- エラー時は `ADMIN_EMAIL`（info@aisjaltd.com）にエラーメールが届きます

## よくある変更

| やりたいこと | 方法 |
|---|---|
| ある媒体を止める | `CONFIG.SEND_TO.SLACK = false` のように該当を false |
| Asanaの送信先ボードを変える | `ASANA_PROJECT_GID` を上表のGIDに変更 |
| Slackの通知文を変える | `notifySlack_()` 内の `text` を編集 |
| 台帳の列を増やす | `appendToSheet_()` のヘッダー行と `appendRow` を編集 |
| WordPressにも送る | `CONFIG.SEND_TO.WORDPRESS = true`（WP側の受け口が必要） |

---

## セキュリティメモ

- Asanaトークン・Slack Webhookは**GAS内にだけ**保存され、フォーム（公開HTML）には一切出ません。安全です。
- 免許証画像は「リンクを知っている人が閲覧可」でDrive保存されます。より厳格にしたい場合は `saveFiles_()` の共有設定を変更してください（例：`ANYONE_WITH_LINK` を外し、担当者にのみ共有）。
