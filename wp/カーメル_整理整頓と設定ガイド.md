# カーメル 整理整頓 ＆ 設定ガイド（やさしいステップ）

非エンジニア向け。**安全第一**：消す前に必ず「OFF（無効化）」で様子を見る。OFFは元に戻せます。

---

## ステップ0：まず「本物」を覚える（これだけ）

この3つが“正”。他の似たものは古いコピーの可能性が高い。

| 役割 | 名前 | どうする |
|---|---|---|
| 記事生成エンジン（本体） | **CARMEL統合管理プラグイン v5.7** | 残す・触らない |
| 自動生成＋MEO（アドオン） | **CARMEL 自動生成（毎日自動）v6.4** | 残す |
| 記事の保存先 | 投稿タイプ **media_article** | 触らない |

---

## ステップ1：プラグインの確認・整理（中身を見る）

1. 左メニュー「**プラグイン**」を開く
2. **残す**：上の「v5.7」と「自動生成（毎日自動）」が**有効**か確認
3. **二重チェック**：同じ役割のものが2つ無いか
   - 「統合管理」系が2つある → 古い方（例：v5.1）を**無効化**
   - 「自動生成」系のプラグインが2つある → 片方だけ
4. **Google投稿の二重化に注意**
   - 外部プラグイン「**Auto Publish for Google My Business**」と、アドオンの「Google自動投稿」を**両方ONにしない**（投稿が二重になる）。どちらか1つに。

---

## ステップ2：スニペット（WPCode）の整理 ★ここが散らかりの原因

「コードスニペット」を開く。**古い“自動生成のコピー”を止める**のが目的。

### 止める（OFF または 削除）
- **7547「CARMEL自動生成」** … プラグインに引っ越したので **OFF**
- **7272「CARMEL 自動生成（WP-Cron）」** … 古い版。OFF
- **7104「OpenRouterを活用した記事自動投稿の再構築依頼」** … 本体の丸コピー＝危険。OFF

### 見分け方（迷ったら）
スニペットを開いて **⌘F（検索）** で次を探す：
- `carmel_get_plugin_settings` か `carmel_auto_run` が**入っていたら古いコピー**→ OFF
- これらが**無ければ別機能の部品**→ そのまま残す

### 触らない（別機能の“部品”。止めると別の所が壊れる）
- 7303「カーメルAPIキー設定」
- お問い合わせSlack通知 / 審査フォーム送信ハンドラ / 会員管理 / 買取写真UI / フッター 等

> ⚠️ ルール：**「自動生成・統合管理のコピー」だけを止める。** 部品スニペットは残す。

## ステップ3：設定で機能を充実（順番どおり）

### 3-1. 設定（最初の1回）
- カーメル管理 → **設定** → **OpenRouter APIキー**を保存（これが無いと記事が作れない）
- ついでに **ブランド設定**（画像の人物像・背景・NG）を整える＝画像の質が上がる

### 3-2. かんたんホームで状態確認
- カーメル管理 → **かんたんホーム**
- 「記事生成エンジン＝接続OK」「APIキー＝設定済み」になっていればOK

### 3-3. 自動生成の設定（テスト→本番）
カーメル管理 → **自動生成**
1. **テーマキュー**を入れる（1行1テーマ）
   ```
   main | special | 自社ローン 審査不安 | 大阪府 | 大阪市 |
   ```
2. **CTA自動修正＝ON**（既定ON）
3. （任意）**画像も生成＝ON** / **本文セクション画像＝ON**
4. まず「**今すぐ1件だけ生成**」を押す → 「メディア記事」に**下書き**ができるか確認
5. 中身OKなら本番へ：**頻度＝毎日** / **自動生成を有効＝ON** / 公開したいなら **生成後すぐ公開＝ON** → 保存

### 3-4. Google投稿（使う場合だけ）
- アドオンの「Google自動投稿」か、外部プラグインの**どちらか片方だけ**ON

### 3-5. MEO（その他掲載ページ）
- カーメル管理 → **その他掲載ページ（MEO対策）**
- 各サイトの**状況（未対応/登録済み/要更新）**とメモを埋めて「保存」
- まず **Googleビジネスプロフィール**から登録・整備（MEOの土台）

## ステップ4：毎日きちんと動かす（仕上げ）

- WP-Cronは「サイトにアクセスがあった時」に動くので、アクセスが少ないと遅れることがある
- 確実にするなら、サーバーの**本物のcron**で1日1回これを叩く：
  ```
  https://carmelonline.jp/wp-cron.php?doing_wp_cron
  ```
- 不安なら、まずは手動の「今すぐ1件だけ生成」で運用してもOK
