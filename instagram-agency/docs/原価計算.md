## 損益分岐

**Light 単独運用の場合**：15,000 ÷ 3,200 ≒ **5社**で分岐
**Standard 単独運用の場合**：15,000 ÷ 7,100 ≒ **3社**で分岐
**Pro 単独運用の場合**：15,000 ÷ 7,700 ≒ **2社**で分岐

一般的な想定ミックス（Light:Standard:Pro = 5:4:1）では、
**約4社**契約すれば固定費を回収できる。

## AI活用による更なる原価低減の余地

| 施策 | 効果 |
|------|-----|
| プロンプトテンプレを業種別・ブランド別に資産化 | 生成品質向上→人チェック工数 40%削減 |
| 分析レポートのHTML自動生成（Notion API + Claude）| レポート工数 60→10分 |
| 予約投稿の一括CSV流し込み | 1投稿1分 → 15秒 |
| Meta Graph API連携で数値自動取得 | レポート原価 -¥1,000/月 |
| 顧客との定型やり取りのFAQ化 | 顧客対応工数 30%削減 |

**上記実装で、Standard粗利率は 36% → 55% まで改善可能**。
価格据え置きで粗利を伸ばす方針が現実的。

## リスクとしての原価変動

| 変数 | 悪化シナリオ | 影響 |
|------|-------------|-----|
| AI API料金値上げ | 2倍 | 1社あたり +¥400 程度、粗利率5pt低下 |
| 為替 円安加速 | 1USD=200円 | 上記に加え +30% |
| 外注チェッカー単価上昇 | ¥2,000/h | Standard粗利率 -8pt |
| Meta APIの制限強化 | 手作業増 | 予約投稿原価 2〜3倍化の可能性 |

これらすべて同時発生でもStandardは粗利率20%を維持できる価格設定。
Lightは**Standardへの入口として位置づけ**、単独では拡大しない設計とする。
