# オークション代行LP × 自動化（GAS＋シート＋LINE）セットアップ手順

LP の「会員登録」「かんたんオーダー」「お問い合わせ」を、
**スプレッドシート登録 → LINE通知 → AI要約** まで自動化する手順です。
GASが初めてでも、画面操作どおりに進めれば動きます（所要 約20分）。

---

## 全体の流れ

```
LP（site/） ──POST──▶ GASウェブアプリ(doPost)
                         ├─ スプレッドシートに登録（オーダー/会員/問い合わせ）
                         ├─ OpenRouterで要約・優先度づけ（任意）
                         └─ LINEでスタッフへ通知（任意）
```

---

## STEP 1：GASプロジェクトを作る

1. https://script.google.com を開き「新しいプロジェクト」。
2. 既定の `コード.gs` の中身を消し、この `gas/` フォルダの各ファイルを
   **ファイルごとに**コピーして貼り付けます（＋でファイル追加）。
   - `Config.gs` / `Setup.gs` / `WebApp.gs` / `Line.gs` / `AI.gs`
3. 保存（💾）。

## STEP 2：シートを自動作成

1. 関数の選択（上部プルダウン）で **`setupAll`** を選び「実行」。
2. 初回は権限の確認 → 「許可」。
3. 実行ログに出る **スプレッドシートID** を、`Config.gs` の
   `SPREADSHEET_ID` に貼り付けて保存。
   （※ 自分で作ったシートを使う場合は、そのIDを入れてから `setupAll` 実行）

これで「オーダー管理 / 会員マスタ / 問い合わせ」の3シートができます。

## STEP 3：ウェブアプリとして公開

1. 右上「デプロイ」→「新しいデプロイ」。
2. 種類＝**ウェブアプリ**。
3. 実行ユーザー＝**自分** ／ アクセスできるユーザー＝**全員**。
4. 「デプロイ」→ 表示される **ウェブアプリのURL**（`.../exec`）をコピー。

## STEP 4：LPと接続

`site/assets/js/app.js` の先頭付近：

```js
var ENDPOINT = ""; // ← ここに STEP3 のURLを貼り付け
```

保存してLPを開き、会員登録やオーダーを実行 → シートに行が増えればOK。

> 補足：GASウェブアプリはCORSヘッダを返さないため、LPからは
> `no-cors`（送信のみ）で送っています。画面表示はLP側で完結し、
> 受信確認はスプレッドシートで行ってください。

## STEP 5：LINE通知（任意）

1. LINE Developers でMessaging APIチャネルを作成。
2. **チャネルアクセストークン（長期）** を `Config.gs` の
   `LINE_CHANNEL_ACCESS_TOKEN` に。
3. 通知を受けたいスタッフの **userId**（または groupId）を
   `LINE_STAFF_IDS` に（カンマ区切りで複数可）。
4. `testLine` を実行して通知が届けば成功。

## STEP 6：AI要約（任意）

1. OpenRouter の APIキーを `Config.gs` の `OPENROUTER_API_KEY` に。
2. 以降、オーダー受付時に申込内容を1〜2文で要約し、
   優先度（高/中/低）を自動付与します。
   （キー未設定でも、予算ベースの簡易ルールで優先度を付けます）

---

## コードを変更したら

GASの仕様上、コード変更を反映するには
「デプロイ」→「デプロイを管理」→ 既存デプロイの ✏️ →
**バージョン＝新しいバージョン** → 保存、が必要です（URLは変わりません）。

## 受信データ（doPostのtype別）

| type | 用途 | 主な項目 |
|------|------|----------|
| `register` | 会員登録 | name, email, plan, source |
| `order` | かんたんオーダー | name, car, bid, cls/clsLabel, fee, total, regionLabel, optionsLabel |
| `contact` | お問い合わせ | name, email, phone, message |
