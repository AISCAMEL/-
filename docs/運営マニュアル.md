# 猫グッズ 無在庫販売 運営マニュアル

**対象:** BASE × Alibaba.com / THE CKB を使った猫グッズの無在庫ドロップシッピング運営
**前提:** 本システム（Hub）で市場調査〜出品〜受注〜発注を半自動化する

---

## 0. 全体像（運営サイクル）

```
【準備】一度だけ
  アカウント開設・API設定
      │
      ▼
【週次】リサーチ → 選定 → 出品   ← 商品ラインナップを増やす
      │
      ▼
【日次】受注 → 自動発注 → 顧客連絡   ← 売れたら回す
      │
      ▼
【自動/日次】在庫・価格同期・損益確認   ← 欠品/赤字を防ぐ
      │
      ▼
【月次】損益分析・カテゴリ見直し
```

無在庫なので「**先に出品 → 売れてから仕入れる**」が基本。在庫リスクはゼロだが、**仕入れ代金は注文発生後すぐ先払い**になるため、運転資金（クレジットカード枠）は必要。

---

## 1. 準備フェーズ（最初に一度だけ）

### 1.1 アカウント・契約
| 項目 | 内容 | 費用 |
|------|------|------|
| BASE 出店 | ショップ開設・**特定商取引法の表記**・ロゴ・配送設定 | 無料（販売時に手数料） |
| THE CKB | 契約・APIキー発行（猫グッズの主力仕入れ先） | プランによる |
| Alibaba.com | アカウント作成（メーカー直仕入れ用） | 無料 |
| 楽天ウェブサービス | `applicationId` 取得（市場調査・無料） | 無料 |

### 1.2 重要な初期設定（無在庫トラブル予防）
- **配送リードタイムの明記**: 中国直送は通常 **2〜4週間**。商品ページと特商法に「お届けまで◯〜◯日」と必ず記載。
- **返品・不良対応ポリシー**: 初期不良の交換/返金条件を明記。
- **価格ルール設定**: 為替バッファ・関税・送料・手数料・目標利益（猫グッズ推奨: **利益率30%以上**）。

### 1.3 システム設定
```bash
cp .env.example .env
# RAKUTEN_APP_ID, THECKB_API_KEY, BASE_*, ALIBABA_* を設定
# CONNECTOR_MODE=live （キー未取得の項目は mock のまま開発可）
```

## 2. 週次オペレーション：リサーチ → 選定 → 出品

### Step 1. 猫グッズを一括リサーチ（システムが自動）
猫グッズのプリセットキーワードで市場調査＋利益判定＋採点を一括実行。

```bash
# 猫グッズのキーワード一覧を取得
curl localhost:3001/niche/cat-goods

# 仕入れ候補を一括スクリーニング（利益率30%以上・B以上だけランキング）
curl -X POST localhost:3001/research/screen \
  -d '{"candidates":[...猫グッズ候補...],"minMarginRate":0.3,"minGrade":"B"}'
```

→ 各候補に **利益・利益率・ROI・スコア(A/B/C)** が付いて返る。

### Step 2. 候補を選ぶ（人の判断）
- **A評価**を中心に採用。利益率・需要（レビュー数）・競合のバランスを確認。
- 規約 block（ペットフード/サプリ/動物用医薬品）は自動除外済み。warn（PSE電気製品・小型部品）は出品可否を人が判断。
- 写真の質・差別化（セット販売・色展開）も加点要素。

### Step 3. 出品（システムが価格計算→BASE登録）
- 価格は自動計算（為替・関税・送料・手数料・利益込み）。
- 商品説明に**配送目安・素材・サイズ・注意書き**を追記（猫の安全に関わる部分は丁寧に）。

**目安: 週に 10〜30 商品ずつ追加し、売れ筋を見ながら絞り込む。**

---

## 3. 日次オペレーション：受注 → 自動発注 → 顧客連絡

```
BASEで注文発生
  → システムが注文を取込（Order）
  → 利益・在庫を最終チェック
  → 仕入れ先（THE CKB/Alibaba）へ自動発注（SupplierOrder）
  → 顧客へ「発送準備中・お届け目安」連絡
  → 追跡番号を取り込み、顧客へ通知
```

- **発注は自動だが、最初は手動承認ステップを挟むと安全**（誤発注・赤字発注の防止）。
- 顧客対応（問い合わせ・遅延連絡）は人が担当。**先回りの連絡がクレームを防ぐ**。

---

## 4. 自動・日次の保守：在庫/価格同期

システムが定期的に仕入れ先を巡回（例: 1時間毎）：
- **在庫0 → BASE 自動非公開**（売れない欠品の販売を防止）
- **仕入れ値・為替の変動 → 価格を自動再計算・更新**
- **在庫復活 → 自動再公開**

→ 「売れたのに仕入れられない」「気づいたら赤字」を自動で防ぐ。

---

## 5. 月次：損益分析・見直し

- カテゴリ別の売上・利益・回転を確認（損益ダッシュボード）。
- **売れ筋カテゴリに寄せる / 不採算商品を撤去**。
- 為替・送料の変化を価格ルールへ反映。

---

## 6. 運営の役割分担（自動 vs 人）

| 工程 | 自動（システム） | 人の判断 |
|------|----------------|----------|
| リサーチ・採点 | ◎ 全自動 | 最終的な採用判断 |
| 価格計算 | ◎ 全自動 | ルール設定 |
| 出品 | ◎ 登録自動 | 写真・説明の質 |
| 受注→発注 | ◎ 自動（承認可） | 異常時の確認 |
| 在庫・価格同期 | ◎ 全自動 | — |
| 顧客対応 | △ 通知自動 | 問い合わせ・クレーム |

---

## 7. リスクと対策（無在庫×中国輸入の現実）

| リスク | 対策 |
|--------|------|
| **配送が遅い**（2〜4週間）→ クレーム | 納期を明記、発送目安を先回り連絡、人気品は国内中継倉庫も検討 |
| **不良品・誤発送** | 検品付きの THE CKB を優先、返金/交換ポリシー整備 |
| **欠品** | 在庫同期で自動非公開、複数仕入れ先を確保 |
| **為替変動で赤字** | 為替バッファ・最低利益ガード、定期同期で価格更新 |
| **規約・法令違反** | 公式API中心、出品前バリデーション、ペットフード/医薬品は扱わない |
| **キャッシュフロー** | 仕入れはカード払いで回す、入金サイクルを把握 |

---

## 8. 立ち上げ初月の進め方（推奨ステップ）

1. **1週目**: BASE開設・特商法/配送ポリシー整備、楽天 `applicationId` 取得、システム設定
2. **2週目**: 猫グッズを一括スクリーニング → A評価から 20〜30 商品を出品
3. **3週目**: 受注対応フローを手動承認で運用、顧客連絡テンプレ整備
4. **4週目**: 売れ筋を分析、価格ルール調整、出品を追加。発注の自動化レベルを上げる

> まずは「**少数を丁寧に**」。回り出したら出品数とautomation を増やす。
