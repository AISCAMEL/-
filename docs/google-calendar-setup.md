# Googleカレンダー連携セットアップ

予約管理画面で「スタッフの予定（Googleカレンダー）」と「店内の予約」を突き合わせ、
**重複しない空き枠だけ**を提示・予約できるようにするための接続手順です。

未連携でも「店内の予約どうしの重複防止」は動きます。スタッフ個人の予定も避けたい場合に連携してください。

---

## 仕組み（概要）

- 空き枠検索時に Google の **FreeBusy API** で予定の埋まり時間を取得し、営業時間から差し引きます。
- 予約確定時に Google カレンダーへ **イベントを作成**します（取消時は削除）。
- 認証は **OAuth2 のリフレッシュトークン**を使います（テナントごとに保存）。

必要なもの：
1. 運営側（あなた）の **Google OAuth クライアント**（`GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`）
2. 各店舗の **カレンダーID** と **リフレッシュトークン**

---

## ステップA：OAuthクライアントを作る（運営側で1回だけ）

1. https://console.cloud.google.com/ でプロジェクトを作成
2. 「APIとサービス」→「ライブラリ」→ **Google Calendar API** を有効化
3. 「OAuth同意画面」を設定（テスト/外部）。スコープに `https://www.googleapis.com/auth/calendar` を追加
4. 「認証情報」→「OAuthクライアントID」→ 種類は **ウェブアプリケーション**
   - リダイレクトURIに `https://developers.google.com/oauthplayground` を追加（トークン取得用）
5. 発行された **クライアントID / クライアントシークレット** を控える
6. Render のバックエンドに環境変数を追加：

| Key | Value |
|---|---|
| `GOOGLE_CLIENT_ID` | `xxxx.apps.googleusercontent.com` |
| `GOOGLE_CLIENT_SECRET` | `GOCSPX-...` |

---

## ステップB：店舗のリフレッシュトークンを取得

各店舗（連携したいGoogleアカウント）でこの作業をします。

1. https://developers.google.com/oauthplayground を開く
2. 右上の歯車 ⚙ →「**Use your own OAuth credentials**」にチェック → 手順AのクライアントID/シークレットを入力
3. 左で `Calendar API v3` →
   `https://www.googleapis.com/auth/calendar` を選び **Authorize APIs**
4. 連携したい店舗のGoogleアカウントでログイン・許可
5. **Exchange authorization code for tokens** を押す
6. 表示された **Refresh token**（`1//...`）をコピー

> カレンダーIDは、Googleカレンダーの「設定」→対象カレンダー→「カレンダーの統合」にある
> **カレンダーID**（個人なら自分のGmailアドレス、共有用なら `xxxx@group.calendar.google.com`）。

---

## ステップC：管理画面で接続

1. 管理画面 →「**予約管理**」→「**Google連携設定**」
2. 入力して保存：
   - **カレンダーID**：手順BのカレンダーID
   - **リフレッシュトークン**：手順Bの `1//...`
   - **1件あたりの所要時間**：査定なら45分など
3. バナーが「✅ Googleカレンダー連携中」になれば成功

---

## 動作確認

1. 「予約管理」→ 希望日を選び「空き枠を表示」
2. Googleカレンダーに予定がある時間帯が**候補から消えている**ことを確認
3. 空き枠を選んで予約 → Googleカレンダー側にもイベントが作られていれば連携OK

---

## よくある質問

| 質問 | 回答 |
|---|---|
| 連携しないと予約できない？ | いいえ。未連携でも店内予約どうしの重複は防げます。Google予定も避けたいときだけ連携 |
| 複数スタッフの予定を見たい | 共有カレンダーを1つ作り、スタッフ予定をそこに集約してそのIDを登録するのが簡単です |
| トークンが無効になった | 同意画面が「テスト」だと期限切れが起きやすい。本番公開にすると安定します。再取得して再保存でOK |
| タイムゾーン | 予約は日本時間(Asia/Tokyo)で扱います |

---

## セキュリティ注意

リフレッシュトークンは**カレンダーへのアクセス権**を持つ秘密情報です。第三者に共有しないでください。
DBに保存されますが、運用ではアクセス権限の管理にご注意ください。
