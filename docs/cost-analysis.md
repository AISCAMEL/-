# 経費・原価分析（AIオペレーター24）

> 結論：**現行プラン（月額固定＋分数上限＋超過課金）は粗利67〜87%で健全。** 採算上の主リスクは
> ①人間転送（携帯への発信レッグ）②高品質TTS音声 ③円安（原価はUSD建て）の3点。
> いずれも「転送別課金・標準音声・上限/超過」で吸収可能。

最終更新：2026-06-11

---

## 1. 前提（為替・調査時点）

- 為替：**1 USD = 155円**（変動するため要更新）。
- 原価はほぼUSD建て、売上はJPY建て → **円安は原価増要因**。
- AIオペレーター24は **STT/TTSをTwilio Conversation Relay側で処理**し、LLMはテキストのみOpenAIに投げる設計。
  → LLM音声API（高額）を使わないため、AIコストは非常に小さい（後述）。

## 3. 1通話あたりの原価（AI完結・転送なし）

LLMトークンの試算（1通話3分・8〜12ターン、システムプロンプト約1,200トークンを毎ターン送信）：
- 入力 約5万トークン × $0.15/1M ≈ **$0.0075**
- 出力 約3千トークン × $0.60/1M ≈ **$0.0018**
- 通話後の要約 1回 ≈ **$0.0005**
- **LLM合計 ≈ $0.01/通話（約¥1.5）← ほぼ無視できる**

**3分のAI通話の原価：**

| 項目 | 計算 | USD | 円 |
|------|------|-----|----|
| Conversation Relay | 3分 × $0.07 | $0.210 | ¥32.6 |
| 着信 | 3分 × $0.0085 | $0.026 | ¥4.0 |
| LLM | （上記） | $0.010 | ¥1.5 |
| **合計** | | **≈$0.246** | **≈¥38（約¥13/分）** |

→ **AI完結通話の変動原価は約¥13/分**（為替¥155前提・標準音声）。

## 4. 転送が入る場合の原価（要注意）

転送中は「着信レッグ継続＋転送先への発信レッグ」が同時に走る（Relay/LLMは停止）。

例：AI 3分 ＋ 担当者と5分通話（転送先＝携帯）
| 区間 | 原価 |
|------|------|
| AI部分(3分) | ¥38 |
| 転送中(5分)：着信継続 5×¥1.3 | ¥6.5 |
| 転送中(5分)：携帯への発信 5×¥12 | ¥60 |
| **合計(8分)** | **≈¥105** |

→ 転送先が**携帯**だと発信レッグで原価が跳ねる（約¥12〜16/分）。
**対策：転送通話は別課金 or プラン分数とは別カウント**（仕様書16章の方針どおり）。固定電話転送なら影響小。

## 5. プラン別 採算（上限まで使った最悪ケース）

変動原価¥13/分・AI完結前提。

| プラン | 月額 | 上限 | 上限時の変動原価 | 粗利(変動) | 粗利率 | 超過単価 vs 原価 |
|--------|------|------|-----------------|-----------|--------|------------------|
| Starter | ¥9,800 | 100分 | ¥1,300 | **¥8,500** | **87%** | ¥80/分 vs ¥13 → +¥67 |
| Business | ¥29,800 | 500分 | ¥6,500 | **¥23,300** | **78%** | ¥60/分 vs ¥13 → +¥47 |
| Pro | ¥59,800 | 1,500分 | ¥19,500 | **¥40,300** | **67%** | ¥50/分 vs ¥13 → +¥37 |

- **超過課金（¥50〜80/分）は原価¥13/分を大きく上回り安全**。使えば使うほど黒字。
- 実際は上限を使い切らない契約が多く、平均粗利はさらに高い。

## 7. 採算リスクと対策（＝「経費的な問題」の核心）

| リスク | 内容 | 対策 |
|--------|------|------|
| **① 人間転送（携帯）** | 発信レッグ¥12〜16/分が原価増 | 転送を**別課金**／転送先は固定電話を推奨／Business以上限定 |
| **② 高品質TTS音声** | ElevenLabs等プレミアム音声はRelay料金が2〜3倍になりうる | 安価プランは**標準音声**、Proのみ高品質を選択制に |
| **③ 円安(FX)** | 原価USD・売上JPY。¥155→¥175で原価+13% | 価格に為替バッファ／年1回見直し条項 |
| **④ LLM設計変更** | OpenAI Realtime音声APIに変えると原価が桁違いに増 | **現行のRelay-STT/TTS＋テキストLLM設計を維持**（重要） |
| **⑤ 長時間通話・いたずら** | 上限なしだと青天井 | **分数上限＋超過課金**（実装済み方針）。1通話の最大長も制限可 |
| **⑥ 録音保存** | 録音ON時はストレージ・Twilio録音料が追加 | 録音はオプション課金／保持期間を設定 |

## 8. 結論

- **価格設計に経費的な問題はない。** 粗利67〜87%、超過課金も原価の4〜6倍で安全。
- 最重要は **「Conversation RelayのテキストLLM設計を崩さない」「転送（特に携帯）を別管理する」** こと。
- 次アクション：
  1. Twilio Console で**日本の着信・番号・発信の実レート**を確認し本表を更新。
  2. 転送通話の課金方針を確定（別課金 or 分数別カウント）。
  3. 為替前提（¥155）をレビューし価格にバッファを織り込む。
  4. ✅ **実装済み**：通話確定時に `usage_records`（`ai_minutes` / `transfer_minutes`）へ原価を記録し、
     管理画面「利用状況・原価」ページ（`/usage`）と API `GET /api/usage`・`GET /api/admin/usage` で
     月次の分数・原価・推定売上・粗利を可視化。レートは `backend/src/billing/rates.ts`（環境変数で改定可）。
