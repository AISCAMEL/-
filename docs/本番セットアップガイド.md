# AUC-AGENT 本番セットアップガイド

合同会社アイズ ／ 最終更新：2026-07-02

このガイドでは、AUC-AGENTシステムを本番環境で稼働させるための設定手順を説明します。

---

## 1. Firebase プロジェクトの設定

### 1-1. プロジェクト作成
1. [Firebase コンソール](https://console.firebase.google.com/) にログイン
2. 「プロジェクトを追加」→ プロジェクト名「auc-agent」を入力
3. Google Analytics は任意（GA4測定IDがあれば有効化推奨）

### 1-2. Authentication 設定
1. 左メニュー「Authentication」→「始める」
2. 「Sign-in method」タブで以下を有効化：
   - **メール/パスワード**: 有効にする
   - **Google**: 有効にする → プロジェクトの公開名とサポートメール（info@aisjaltd.com）を入力

### 1-3. Cloud Firestore 設定
1. 左メニュー「Firestore Database」→「データベースを作成」
2. 「本番モード」を選択（セキュリティルール設定後）
3. ロケーション: `asia-northeast1`（東京）推奨

### 1-4. Firestore セキュリティルール
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /members/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
      match /orders/{orderId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
      match /quotes/{quoteId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
      match /coupons/{couponId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
    }
  }
}
```

### 1-5. 設定値の反映
1. Firebase コンソール → プロジェクト設定（歯車アイコン）→「全般」
2. 「マイアプリ」→ ウェブアプリを追加（まだの場合）
3. 表示される `firebaseConfig` の各値をコピー
4. `site/assets/js/firebase-config.js` を編集し、プレースホルダーを置換：

```javascript
var firebaseConfig = {
  apiKey: '実際のAPIキー',
  authDomain: 'auc-agent.firebaseapp.com',
  projectId: 'auc-agent',
  storageBucket: 'auc-agent.appspot.com',
  messagingSenderId: '123456789',
  appId: '1:123456789:web:abcdef'
};
```

## 2. Google Apps Script（GAS）のデプロイ

### 2-1. スプレッドシート作成
1. Google ドライブで新しいスプレッドシートを作成
2. 「拡張機能」→「Apps Script」を開く

### 2-2. GASファイルの配置
`gas/` フォルダの全ファイルをApps Scriptプロジェクトにコピー：
- `Config.gs` — 設定値（最初に編集）
- `WebApp.gs` — Webアプリ入口
- `Setup.gs` — シート初期化
- `Line.gs` — LINE/顧客通知
- `Slack.gs` — Slack連携
- `AI.gs` — AI要約
- `Dashboard.gs` — ダッシュボード集計

### 2-3. Config.gs の設定
```javascript
SPREADSHEET_ID: "スプレッドシートのID（URLの/d/と/editの間の文字列）",
LINE_CHANNEL_ACCESS_TOKEN: "LINE Developersで取得",
LINE_STAFF_IDS: "スタッフのuserId（カンマ区切り）",
ANTHROPIC_API_KEY: "Claude APIキー（任意）",
SLACK_WEBHOOK_URL: "Slack Incoming WebhookのURL（任意）",
SLACK_SLASH_TOKEN: "Slackスラッシュコマンドの検証トークン（任意）",
```

### 2-4. 初期セットアップ実行
1. Apps Scriptエディタで `setupAll` 関数を選択し「実行」
2. 初回は認証の承認が求められるので許可する
3. スプレッドシートに7つのシートが自動作成される：
   - オーダー管理、出品管理、ローン審査、会員マスタ、問い合わせ、相場見積り、紹介管理

### 2-5. ウェブアプリとしてデプロイ
1. 「デプロイ」→「新しいデプロイ」
2. 種類: ウェブアプリ
3. 実行ユーザー: 自分 ／ アクセスできるユーザー: 全員
4. 「デプロイ」をクリック
5. 表示される **ウェブアプリのURL** をコピー

### 2-6. エンドポイントの接続
`site/assets/js/app.js` を編集：
```javascript
var ENDPOINT = "https://script.google.com/macros/s/XXXX/exec";
```
上記のXXXXをデプロイ時に取得したURLに置換。

### 2-7. 週次レポートのトリガー設定（任意）
1. Apps Scriptエディタ → 左メニュー「トリガー」
2. 「トリガーを追加」
3. 関数: `sendWeeklyReport` / 時間主導型 / 週ベース / 毎週月曜 9:00

## 3. LINE公式アカウント設定（任意）

### 3-1. Messaging API チャネル作成
1. [LINE Developers](https://developers.line.biz/) にログイン
2. プロバイダー作成 → 新規チャネル「Messaging API」
3. チャネルアクセストークン（長期）を発行 → Config.gs の `LINE_CHANNEL_ACCESS_TOKEN` に設定

### 3-2. スタッフ通知先の設定
- 個人通知: 各スタッフのuserIdをカンマ区切りで `LINE_STAFF_IDS` に設定
- グループ通知: グループIDを1つだけ設定

### 3-3. 顧客通知
顧客向けLINE通知は、会員マスタの「LINE_UID」列にユーザーのLINE IDが登録されている場合に自動送信されます。LINE友だち追加時のWebhookでuidを取得し、メールアドレスと紐付けてください。

## 4. Slack連携設定（任意）

### 4-1. Incoming Webhook
1. [Slack App](https://api.slack.com/apps) で新規アプリ作成
2. 「Incoming Webhooks」を有効化 → WebhookURLを取得
3. Config.gs の `SLACK_WEBHOOK_URL` に設定

### 4-2. スラッシュコマンド
1. 「Slash Commands」で以下を追加：
   - `/相場回答` — Request URL: GASウェブアプリURL
   - `/進捗` — 同上
   - `/ローン` — 同上
2. 「Verification Token」を Config.gs の `SLACK_SLASH_TOKEN` に設定

## 5. Vercelへのデプロイ

### 5-1. 初回デプロイ
1. [Vercel](https://vercel.com/) にGitHubアカウントでログイン
2. 「Import Project」→ このリポジトリを選択
3. **Root Directory**: `site` に設定（vercel.jsonが自動認識される）
4. 「Deploy」をクリック

### 5-2. カスタムドメイン設定
1. Vercelプロジェクト → 「Settings」→「Domains」
2. `auc-agent.com`（または任意のドメイン）を追加
3. DNSレコードを設定：
   - タイプ: CNAME / ホスト: @ / 値: cname.vercel-dns.com
   - または Aレコード: 76.76.21.21

### 5-3. 環境確認
vercel.json により以下が自動設定済み：
- クリーンURL（.html省略）
- セキュリティヘッダー（CSP, X-Frame-Options等）
- 静的アセット1年キャッシュ
- Firebase CDNドメイン許可

## 6. GA4（Google Analytics 4）設定（任意）

1. [Google Analytics](https://analytics.google.com/) で新しいプロパティを作成
2. ウェブストリームを追加し「測定ID」（G-XXXXXXXXXX）を取得
3. 各HTMLファイルの `<!-- GA4 -->` コメント付近の測定IDを置換

---

## 7. メール送信のドメイン認証

GASからのメール送信（オリコ申込書送付、相場回答通知等）の到達率を高めるため：
1. Google Workspace でカスタムドメインメール（info@aisjaltd.com）を設定
2. DNS に SPF レコードを追加: `v=spf1 include:_spf.google.com ~all`
3. DKIM を有効化（Google Workspace管理画面から）
