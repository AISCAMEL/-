# AUC-AGENT 公開（デプロイ）手順書

このサイトは**静的サイト（HTML/CSS/JS）**なので、無料の静的ホスティングにそのまま公開できます。
申込の自動化（スプレッドシート登録・LINE通知・AI要約）は **GAS（Google Apps Script）** を別途デプロイして接続します。

---

## 0. 全体像
```
[公開サイト（site/）] ──申込──▶ [GAS Web App] ──▶ スプレッドシート登録 / LINE通知 / AI要約
   Netlify / Cloudflare Pages / GitHub Pages 等         （gas/ をデプロイ）
```
購入のやりとりは外部「車ひろば エージェント」（agent.car-hiroba.jp）を利用。

---

## A. サイトを公開する（いずれか1つ）

### A-1. Netlify（ドラッグ&ドロップが最速）
1. https://app.netlify.com/ に登録・ログイン
2. 「Add new site」→「Deploy manually」
3. `site/` フォルダを**そのままドラッグ&ドロップ**
4. 発行されたURL（例 `https://xxxx.netlify.app`）で即公開。独自ドメインは「Domain settings」から設定。

### A-2. Cloudflare Pages（GitHub連携）
1. このリポジトリ（`aiscamel/-`）をCloudflare Pagesに接続
2. ビルド設定：**ビルドコマンドなし**／**出力ディレクトリ = `site`**
3. デプロイ → 発行URLで公開。

### A-3. GitHub Pages
1. リポジトリ Settings → Pages
2. Source：対象ブランチ ／ フォルダ `/site`（※Pagesはサブフォルダ指定可。不可なら `site/` の中身をルートに置くか `docs/` 運用）
3. 公開URL：`https://aiscamel.github.io/-/`

> いずれも `index.html` が入口。`sell.html` `shipping.html` `mypage.html` `login.html` `manual.html` `privacy.html` も同階層に揃っています。

## B. 申込の自動化（GAS）を接続する ＝ 実行ガイド

> 所要 約20分。クリック順どおりに進めれば動きます。詳細版は `gas/セットアップ手順.md`。

### B-1. プロジェクト作成とコード貼り付け
1. https://script.google.com/ →「新しいプロジェクト」。
2. 既定の `コード.gs` を消し、`gas/` の各ファイルを**ファイル単位で**追加・貼り付け：
   `Config.gs` / `Setup.gs` / `WebApp.gs` / `Line.gs` / `Slack.gs` / `AI.gs`。
3. 💾 保存。

### B-2. 設定（`Config.gs`）
| キー | 用途 | 必須 |
|---|---|---|
| `SPREADSHEET_ID` | データ保存先。空なら `setupAll` が自動作成 | 任意 |
| `LINE_CHANNEL_ACCESS_TOKEN` / `LINE_STAFF_IDS` | LINE通知 | 任意 |
| `SLACK_WEBHOOK_URL` | Slack通知（Incoming Webhook） | 任意 |
| `SLACK_SLASH_TOKEN` | Slackから `/相場回答` 等を使う場合 | 任意 |
| `OPENROUTER_API_KEY` | 申込のAI要約・優先度づけ | 任意 |
| `SELL_SHEET_PDF_FOLDER_ID` | 出品票・オリコ申込書PDFの保存フォルダ | 任意 |

### B-3. シート作成 → 公開
4. 関数プルダウンで **`setupAll`** を選び「実行」→ 初回は権限を「許可」。
   → 6シート（オーダー管理／出品管理／ローン審査／会員マスタ／問い合わせ／相場見積り）が自動作成。
5. 「デプロイ」→「新しいデプロイ」→ 種類「**ウェブアプリ**」／実行ユーザー「**自分**」／アクセス「**全員**」→ デプロイ。
6. 発行された**ウェブアプリURL（`.../exec`）**をコピー。

### B-4. LPと接続
7. `site/assets/js/app.js` 先頭の `ENDPOINT` に貼り付けて再公開：
   ```js
   var ENDPOINT = "https://script.google.com/macros/s/XXXX/exec";
   ```
   → 会員登録・購入オーダー・出品申込・ローン仮申込・相場見積り・お問い合わせが、
   自動でシート登録＋LINE/Slack通知＋（出品票/オリコ申込書）PDF生成されます。
8. **BUYMOサイトも接続する場合**：BUYMOの公開手順は次章 **C2** にまとめています
   （査定・問い合わせ・パートナー応募は `type:"buymo"` で送信。空欄ならデモ動作）。

### B-5. 動作確認（接続できたかのチェック）
- LPで会員登録 → **会員マスタ**シートに行が増える。
- 購入オーダー送信 → **オーダー管理**に行が増え、LINE/Slackに通知が届く。
- ローン仮申込 → 入力メール宛に**オリコ申込書PDF**が届く。
- Slackで `/相場回答 QT-xxxx 金額` → **相場見積り**シートが「回答済み」、マイページに反映。

> ⚠️ コードを直したら反映には再デプロイが必要：
> 「デプロイ」→「デプロイを管理」→ ✏️ →「バージョン＝新しいバージョン」→ 保存（URLは不変）。

## C2. BUYMO（車買取サイト）公開手順 ★このブランチの主対象

BUYMO は `site/` 内の静的サイト（トップLP・25ジャンルLP・64掛け合わせLP・47都道府県LP・各ハブ・問い合わせ/サンクス・業務システム）。
ホスティングはA章と同じ（publish ディレクトリ＝`site`、`netlify.toml`/`_headers` 同梱）。GAS接続はB章と同じ `/exec` を使う。

### C2-1. 本番ドメインを反映して全ページ再生成（重要・1コマンド）
ドメイン確定後、`SITE_URL` を渡して再生成すると **canonical / sitemap.xml / robots.txt が絶対URLに**なります（未指定なら相対のまま）。
```bash
cd site
SITE_URL=https://（本番ドメイン） node tools/gen-genre.js   # ハブ＋25ジャンル＋64掛け合わせ
SITE_URL=https://（本番ドメイン） node tools/gen-area.js    # 47都道府県＋ハブ＋sitemap.xml＋robots.txt
```
> ジャンルや文言を変えたとき（`assets/js/genres.js` / `tools/_cross.js` 等）も上記2コマンドで再生成。

### C2-2. 公開前QA（両方 ✅ を必ず確認）
```bash
node tools/check-links.js   # 内部リンク／アセットの健全性
node tools/seo-check.js     # JSON-LD・title重複・canonical・OGP・データ整合
```
※ push時に GitHub Actions（`.github/workflows/qa.yml`）でも自動実行されます。

### C2-3. フォーム／業務システムを GAS に接続
B章で発行した同じ `.../exec` URL を、以下 **5ファイルの `ENDPOINT`** に設定（空欄ならデモ動作）：

| ファイル | 役割 |
|---|---|
| `assets/js/buymo.js` | 査定・問い合わせ・パートナー応募（`type:"buymo"`／送信成功で `buymo-thanks.html` へ） |
| `assets/js/auth.js` | 業務ログイン（本部/加盟店/会員） |
| `assets/js/hq-common.js` | 案件ボード・リード・加盟店・お知らせの読み書き |
| `assets/js/member.js` | 会員マイページ（査定進捗） |
| `assets/js/report.js` | 営業レポート |

> `board.js`/`hq-leads.js`/`hq-stores.js`/`notices.js` は `hq-common.js` 経由、`academy.js`/`simulator-buy.js`/`chatbot.js`/`genres.js` はクライアント完結で個別設定不要。

### C2-4. 計測（GA4）
`assets/js/analytics.js` の `MEASUREMENT_ID` に `G-XXXXXXXXXX` を設定。
- 主要イベント：`simulate`（査定シミュ実行）／`generate_lead`（フォーム送信）／`conversion`（送信完了ページ到達）
- GA4 管理画面で **`conversion`（または `generate_lead`）をコンバージョン**に設定。サンクスページ `buymo-thanks.html` が着地点。

### C2-5. 認証アカウント・ステップメール
- 本部/加盟店アカウント：GASで `createUser(email, pw, role, name)` を実行（`role` は `hq`/`partner`）。
- ステップメール：GASトリガーで `runStepMails` を日次登録（`gas/ステップメール_セットアップ.md`）。

### C2-6. BUYMO 公開前チェックリスト
