# 本番公開（デプロイ）手順

猫グッズ無在庫販売ハブを実際にネット上で動かすための手順。
構成は **管理画面（Vercel）＋ Hub API（Railway等）** の2つを公開します。

```
ブラウザ ──HTTPS──▶ 管理画面 (Vercel)  ──▶  Hub API (Railway/Render/VPS)  ──▶ 楽天/Yahoo/BASE...
            ログイン必須              HUB_API_URL で接続
```

---

## 0. 事前準備
- GitHub にこのリポジトリがある（済）
- 楽天 `RAKUTEN_APP_ID` / Yahoo `YAHOO_APP_ID`（取得済み）
- 管理画面のログインパスワードを決める

---

## 1. Hub API を公開（Railway / Render / VPS）

リポジトリ直下の `Dockerfile` でそのままデプロイできます。

### Railway の例
1. Railway で New Project → Deploy from GitHub repo → このリポジトリ
2. ビルド方式: **Dockerfile**（自動検出）
3. 環境変数を設定：
   | 変数 | 値 |
   |---|---|
   | `RAKUTEN_APP_ID` | 楽天のアプリID |
   | `YAHOO_APP_ID` | Yahoo の Client ID |
   | `SYNC_INTERVAL_MINUTES` | `60`（定期同期・任意） |
   | `API_PORT` | `3001`（または Railway の `PORT` に合わせる） |
4. デプロイ後の公開URL（例 `https://hub-api.up.railway.app`）を控える

> ⚠️ **楽天IP制限の注意**: 楽天APIは「アプリ登録したIP」からのリクエストのみ許可します。
> Hub API を置くサーバーの**送信元グローバルIP**を、楽天デベロッパーの「許可されたIPアドレス」に登録してください。
> Railway/Render は送信IPが変動する場合があるため、**固定IPのVPS**（例: さくら/ConoHa/Lightsail）に置くと安定します。
> （Yahoo は IP 制限が無いので、まず Yahoo だけ live にして動作確認するのも手です）

---

## 2. 管理画面を公開（Vercel）

1. Vercel で New Project → このリポジトリを Import
2. **Root Directory** を `apps/web` に設定
3. Framework: Next.js（自動検出）/ Install Command: `pnpm install`（モノレポを自動解決）
4. 環境変数を設定：
   | 変数 | 値 |
   |---|---|
   | `HUB_API_URL` | 手順1のHub APIの公開URL |
   | `DASHBOARD_PASSWORD` | ログインパスワード（必須） |
   | `AUTH_TOKEN` | ランダムな秘密文字列（32文字以上推奨） |
5. Deploy → `https://xxxx.vercel.app` でアクセス

→ アクセスすると**ログイン画面**が出ます。`DASHBOARD_PASSWORD` でログイン。

---

## 3. 公開後のセキュリティ確認

- [ ] `DASHBOARD_PASSWORD` を設定した（未設定だと**誰でもアクセス可**）
- [ ] `AUTH_TOKEN` をランダム値にした
- [ ] HTTPS でアクセスできる（Vercel/Railway は自動）
- [ ] APIキー類は環境変数で管理し、コードにハードコードしていない
- [ ] 楽天の許可IPに API サーバーのIPを登録した

---

## 4. ローカルとの違い（まとめ）

| | ローカル | 本番 |
|---|---|---|
| 管理画面 | `localhost:3000` | `https://xxx.vercel.app` |
| Hub API | `localhost:3001` | `https://xxx.railway.app` |
| 認証 | `DASHBOARD_PASSWORD` 未設定なら無し | **必須** |
| 接続 | `HUB_API_URL=localhost:3001` | `HUB_API_URL=本番APIのURL` |

---

## 補足: もっと簡単に試すなら
- **まず管理画面だけ Vercel に公開**し、Hub API はローカルやVPSで動かす構成でも可。
- 本格運用では DB（Supabase/Neon）と キュー（BullMQ/Redis）を足して、出品・受注・同期を永続化します（設計書 §10/§13 参照）。
