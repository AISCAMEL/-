# セキュリティレビュー（本番リリース前最終確認）

対象：`wp-content/plugins/carmel-core`（要件定義 v1.5）
実施日：2026-06-16 ／ 範囲：プラグイン全コード

> 注記：このリポジトリ環境に WordPress 本体はないため、本レビューは**静的コードレビュー＋PHP構文チェック**による確認です。実環境での動的検証（権限・通知・連携・Cron の挙動）は別途必要です。

## 結論

重大な脆弱性は検出されませんでした。WordPress のセキュリティ要件（nonce・権限・エスケープ・サニタイズ・署名検証）は実装済みです。レビューで見つかった**軽微なベストプラクティス不足（nonce値のunslash）を修正**しました。

## チェック項目と結果

| 項目 | 状態 | 内容 |
|------|:----:|------|
| CSRF（nonce） | ✅ | 全 admin-post（審査/契約/店舗/書類/CSV）で `wp_verify_nonce`。AJAX は `check_ajax_referer` |
| 権限（capability） | ✅ | 各ハンドラで `current_user_can`。遷移先別 cap（審査=carmel_screening、契約=carmel_send_contract、店舗操作=carmel_change_deal_status、本部=carmel_manage_stores） |
| 行レベルアクセス制御 | ✅ | 加盟店は自店 `store_id` のみ操作可（店舗ポータル・書類）。顧客は自分の `customer_id` のみ閲覧 |
| 出力エスケープ | ✅ | 画面出力は `esc_html` / `esc_attr` / `esc_url`。バナーは固定文言を `esc_html` |
| 入力サニタイズ | ✅ | `(int)` / `sanitize_key` / `sanitize_text_field` / `sanitize_email`。`wp_unslash` 併用 |
| SQL インジェクション | ✅ | 直接SQLなし。`WP_Query` / `get_posts` / メタAPIのみ |
| ファイルアップロード | ✅ | `wp_check_filetype`（JPG/PNG/PDF）＋サイズ上限＋`wp_handle_upload`（mimes制限） |
| アップロード保存 | ✅ | `uploads/carmel-secure/{deal_id}` に `deny all` の .htaccess を自動設置（直アクセス不可） |
| ファイル配信 | ✅ | 公開URLなし。nonce＋権限＋**secureディレクトリ封じ込め**で `readfile` |
| Webhook 署名/トークン | ✅ | Square＝HMAC-SHA256署名検証、GAS/マネーフォワード契約＝Secret Token、いずれも `hash_equals` |
| REST 申込口 | ✅ | トークン未設定時は既定で拒否（オープン化はフィルタ明示時のみ） |
| 直接ファイルアクセス防止 | ✅ | 全PHPで `defined('ABSPATH') || exit` |
| 最小権限 | ✅ | customer は遷移操作不可（閲覧のみ）、加盟店に本部専用遷移を提示しない＋cap二重ブロック |

## 本レビューでの修正

- **nonce 値の unslash 追加**：`wp_verify_nonce()` へ渡す `$_POST`/`$_GET` の nonce を `sanitize_text_field( wp_unslash( … ) )` 経由に変更（審査・店舗・契約・書類アップロード/ダウンロード・CSV）。マジッククォート環境での検証失敗を防止。

## 運用・本番前の推奨事項（コード外）

1. **HTTPS 必須**（決済・署名・個人情報・トークン送信のため）
2. **連携トークン/キーは `wp-config.php` 定数**で管理（DB保存より露出を抑制）
3. **Nginx 環境**では `.htaccess` が効かないため、`uploads/carmel-secure/` を location ブロックで拒否設定（Apache は .htaccess で対応済み）
4. 個人情報・本人確認書類の**保存期間ポリシー**を確定（要件§13-#2）
5. WordPress 本体・プラグインの定期アップデートと WAF（WPX 提供機能等）
6. 監査ログ（`_carmel_status_history`）の定期バックアップ

## 残課題

- 会費の最終課金方式の確定（§13-#1。現状は WCサブスク／手動の両対応で運用可）
- 書類保存期間ポリシー（§13-#2）
- 実 WordPress 環境での結合・権限・連携の動的テスト
