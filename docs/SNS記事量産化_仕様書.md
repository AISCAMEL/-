# 📣 SNS記事量産化システム 仕様書 v1.0

> CarLoan_System の **Stage 0（集客）** を強化するサブシステム。
> OpenRouter AI を使って、中古車販売・カーローン・車買取・自社リースの SNS 投稿原稿を **テーマ別・媒体別に自動量産** し、スプレッドシートに蓄積・運用する。

最終更新：2026-06-08
親仕様書：[`開発仕様書.md`](./開発仕様書.md)

---

## 1. 目的・位置づけ

| 項目 | 内容 |
|------|------|
| 目的 | SNS集客記事を AI で大量・継続生成し、投稿運用の工数をゼロに近づける |
| 親システム | CarLoan_System（業務フロー Stage 0：集客） |
| 開発言語 | Google Apps Script（JavaScript） |
| AI | OpenRouter API（`deepseek-free → gemini-free → claude-haiku` の段階移行） |
| 保存先 | スプレッドシート `CarLoan_管理DB` に「⑪ SNS記事」シートを追加 |
| 運用形態 | スプレッドシートのカスタムメニュー＋定期トリガーで運用 |

**ランニングコスト**：OpenRouter のみ（既存予算 約500円/月の枠内。無料モデル優先で実質ほぼ0円）

## 2. 対象SNSと記事フォーマット

| 媒体コード | 媒体 | 文字量目安 | 構成 | ハッシュタグ |
|-----------|------|-----------|------|-------------|
| `X` | X (旧Twitter) | 〜140字 | 1メッセージ完結・フック先行 | 2〜3個 |
| `IG` | Instagram | 200〜400字 | 結論→共感→CTA・改行多め | 8〜15個 |
| `TT` | TikTok / YouTube Shorts | 台本形式 | フック(3秒)→本編→CTA | 3〜5個 |
| `BLOG` | LINE配信 / ブログ・LP | 800〜1500字 | 見出し付き・SEO意識・検索流入用 | — |

各媒体は同一テーマから**媒体最適化された別原稿**を生成する（使い回しではなく媒体別に作り分け）。

---

## 3. テーマ設計（記事の量産軸）

中古車販売業の集客で刺さるテーマを「カテゴリ × 切り口」で組み合わせ、量産の母数を確保する。

### 3.1 テーマカテゴリ

| コード | カテゴリ | 想定読者 |
|-------|---------|---------|
| `T01` | カーローン審査（通りやすくするコツ） | 審査に不安がある人 |
| `T02` | 自社ローン・ブラック対応 | 他社で断られた人 |
| `T03` | 車買取・高価買取（2026年相場） | 売却検討者 |
| `T04` | 自社リース（頭金なし・GPS） | 月々払いで乗りたい人 |
| `T05` | 個人事業主・法人向けローン | 経営者・フリーランス |
| `T06` | 中古車選びのコツ・お役立ち | 購入検討の初期層 |
| `T07` | お客様の声・成約実績 | 信頼を求める層 |
| `T08` | 季節・イベント（ボーナス/決算/新生活） | 購入タイミング層 |

### 3.2 切り口（フック）パターン

`質問型` / `数字型` / `失敗談型` / `比較型` / `実績型` / `共感型` / `緊急型` / `お役立ちリスト型`

> **量産の計算式**：8カテゴリ × 8切り口 × 4媒体 = **256通り**の組み合わせを起点に、各組み合わせから複数バリエーションを生成可能。

## 5. GASファイル構成（追加分）

| ファイル | 役割 | 主な関数 |
|---------|------|---------|
| `SnsArticle.gs` | SNS記事の量産・保存・メニュー | `generateSnsArticles()` / `buildSnsPrompt()` / `saveSnsArticle()` / `onOpen()` |
| `SnsConfig.gs` | テーマ・切り口・媒体の定義（マスタ） | `getSnsConfig()` |

> AI呼び出しは親仕様の `OpenRouter.gs` の `callOpenRouter()` を共用する。
> 未実装の場合に備え `SnsArticle.gs` 内に最小限のフォールバック呼び出しを内蔵。

### 主要関数の責務

- `generateSnsArticles(count, platforms, categories)`
  指定件数・媒体・カテゴリで一括生成し、⑪シートに追記。
- `buildSnsPrompt(platform, category, hook)`
  事業情報＋媒体ルール＋テーマ＋切り口を組み込んだプロンプトを構築。
- `saveSnsArticle(row)`
  生成結果を1行整形して書き込み（記事ID自動採番・文字数自動計算）。
- `generateDailySnsBatch()`
  定期トリガー用。毎日 N 件を自動生成（媒体・テーマはローテーション）。

## 6. 自動トリガー設計

| トリガー | タイミング | 関数 | 動作 |
|---------|-----------|------|------|
| 時間主導 | 毎朝 8:00 | `generateDailySnsBatch` | 各媒体1〜2件を自動生成（下書き状態） |
| シート起動 | スプレッドシート開いた時 | `onOpen` | カスタムメニュー「📣 SNS記事」を追加 |

カスタムメニュー項目：

- 「✍️ まとめて生成（件数指定）」
- 「📅 今日のバッチを生成」
- 「🔁 没記事を再生成」

---

## 7. プロンプト設計方針

`buildSnsPrompt()` は次の要素を必ず注入する：

1. **事業コンテキスト**：中古車販売・カーローン・車買取・自社リースのフランチャイズである旨
2. **媒体ルール**：文字数・構成・ハッシュタグ数（§2の表）
3. **テーマ＋切り口**：§3 の組み合わせ
4. **トーン**：親しみやすく信頼感のある日本語。誇大広告・断定的な審査保証は禁止
5. **コンプライアンス制約**：
   - 「必ず審査が通る」等の断定表現を禁止
   - 貸金業・割賦の法令に触れる誇大表現を避ける
   - 個人を特定できる実名・写真の記載をしない
6. **出力形式**：JSON（`title` / `body` / `hashtags`）で返させ、パースして保存

## 8. 運用フロー

```
[自動生成] 毎朝バッチ or 手動メニュー
   ↓ ⑪SNS記事シートに「下書き」で蓄積
[人による確認] 担当者が内容チェック → ステータス「承認済」
   ↓
[投稿] 各SNSへ投稿（手動 or 将来API連携）→「投稿済」に更新
   ↓
[分析] 反応の良いテーマ/切り口を週次レポートに反映 → 次回生成に活用
```

将来拡張：X API / Instagram Graph API / LINE Messaging API による**自動投稿**まで連携可能。

---

## 9. 導入手順（GAS初心者向け）

1. `CarLoan_System` の GAS プロジェクトに `SnsConfig.gs` と `SnsArticle.gs` を新規追加し、全文を貼り付け
2. メニュー「📣 SNS記事 > 🛠 初期セットアップ」を実行 → ⑪シート自動作成
3. OpenRouter の API キーを `Config.gs` の `getConfig()` に設定（未設定でもサンプル生成は可能）
4. メニュー「✍️ まとめて生成」で件数を入力 → 量産開始
5. 生成された下書きを確認・承認し、各SNSへ投稿
