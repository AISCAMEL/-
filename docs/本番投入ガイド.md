# カーメル統合管理システム 本番投入ガイド

`carmel-core` プラグインを実 WordPress に載せ、稼働させるための手順（運用者向け・上から順に実施）。
詳細仕様は `wp-content/plugins/carmel-core/README.md`、テストは `docs/結合テストチェックリスト.md`、
LINE は `docs/LINE_LIFF_セットアップとテスト.md` を参照。

---

## STEP 0. 前提プラグイン

- **必須**：ACF Pro（フィールド表示・編集）
- **決済**：WooCommerce ＋ Square for WooCommerce
- **フォーム**：WPForms（申込/問い合わせ）※CF7 / Gravity / MW WP Form / Snow Monkey でも可
- **任意**：bbPress（外部コミュニティ）／WooCommerce Subscriptions（会費サブスク時）

> 未導入でもプラグインは有効化でき、未設定の連携は安全に no-op（システム全体は動作）。

## STEP 1. プラグイン配置・有効化

1. `wp-content/plugins/carmel-core/` を配置
2. 管理画面で **有効化** → CPT・ロール登録／Cron スケジュール／初期コンテンツ投入／リライトflush が自動実行
3. 既存環境でロール/CPT変更を反映したい時は **一度無効化→再有効化**

| サービス | 定数 / オプション |

| LINE WORKS | `CARMEL_LINEWORKS_WEBHOOK` |
| Slack | `CARMEL_SLACK_WEBHOOK` |
| GAS（AIスコア/PDF） | `CARMEL_GAS_ENDPOINT` / `CARMEL_GAS_TOKEN` |
| Google Maps | `CARMEL_MAPS_API_KEY`（Maps JS＋Geocoding＋Embed 有効化） |
| Square | `CARMEL_SQUARE_SIGNATURE_KEY` / `CARMEL_SQUARE_WEBHOOK_URL` |
| マネーフォワード契約 | `CARMEL_MF_ENDPOINT` / `CARMEL_MF_TOKEN` |
| 申込REST | `CARMEL_INTAKE_TOKEN` |
| LINE公式 Messaging API | `CARMEL_LINE_CHANNEL_TOKEN` / `CARMEL_LINE_CHANNEL_SECRET` / `CARMEL_LINE_MODE`(`proline`→`line`) |
| LIFF | `CARMEL_LIFF_ID` / `CARMEL_LIFF_CHANNEL_ID` |
| Notion / bbPress | `CARMEL_NOTION_URL` / `CARMEL_COMMUNITY_URL` |

**オプション（`wp_options`・運用設定）**：
`carmel_company_info`（帳票の発行元）/ `carmel_tax_rate`（税率10）/ `carmel_commission_rate`（手数料率5）/
`carmel_finance_defaults`（年率・回数・残価）/ `carmel_warranty_plans`（保証プラン）/
`carmel_member_page_url`（会員ページ=会員ログインLIFFのURL）/ `carmel_line_form_url`（審査LIFF URL）/
`carmel_line_ai_endpoint`（AI応答先）/ `carmel_lead_sla_hours`（反響SLA 24）/ `carmel_deal_stale_days`（商談停滞 7）。

## STEP 4. 外部 webhook の登録

- Square → `https://例.jp/wp-json/carmel/v1/square-webhook`
- GAS → `https://例.jp/wp-json/carmel/v1/gas-callback`
- マネーフォワード契約 → `https://例.jp/wp-json/carmel/v1/mf-contract-callback`

## STEP 5. フォーム連携（WPForms）

`functions.php` か mu-plugin に：

```php
// 取り込む WPForms フォームID（=投稿ID）
add_filter( 'carmel_wpforms_forms',         fn() => array( 7348, 7361 ) );
// 反響扱い（スコア無し）にするフォーム
add_filter( 'carmel_wpforms_inquiry_forms', fn() => array( 7361 ) );
// LINE反響→担当加盟店のエリア割当（エリア => carmel_store のID）
add_filter( 'carmel_area_store_map', fn() => array(
    '関東' => 12, '近畿' => 18, '中部' => 20, '九州・沖縄' => 25,
) );
```

→ 7348＝審査申込（AIスコア対象）/ 7361＝お問い合わせ（反響）。フィールドは型・ラベルから自動マッピング。

## STEP 6. ユーザー・データの初期設定

- テストユーザー：`hq_admin` / `store_owner`（user_meta `store_id` 付）/ `store_staff`（`store_id` 付）/ `customer`
- `carmel_store`（加盟店）を作成：店舗名・住所・電話・営業時間・エリア・取扱種別・オーナーID
- `carmel_vehicle`（在庫）を登録（CSV一括取込・`image_urls`で画像も可）→「掲載（共有）」で公開
- LINE ユーザーは user_meta `line_user_id`（LIFFで自動取得）

## STEP 7. 最小動作確認（スモークテスト）

1. `/apply` から申込 → 顧客アカウント自動発行＋案件作成＋受付通知（GAS設定時はAIスコア→`scored`）
2. `/hq` の `[carmel_hq_screening]` で審査OK → 顧客へ通知、`/mypage` に反映
3. `/store` で工程前進・顧客ひも付け・次アクション設定 → 「今日のやること」「履歴」確認
4. `/inventory` 公開・絞り込み・詳細（月々ウィジェット）・お気に入り
5. `/stores` 店舗ページ・レビュー（`[carmel_hq_reviews]` で承認）
6. LINE：友だち追加→自動応答→審査LIFF→`line_user_id`保存→通知がLINEへ

## STEP 8. 結合テスト → 本番

- `docs/結合テストチェックリスト.md`（全22節）を実機で実施
- セキュリティ最終確認（直近レビューで高リスク0・比較ページIDOR修正済み）
- バックアップ取得 → 本番公開

## 段階移行・運用メモ

- **通知**：当面 `carmel_line_mode=proline`。LINE公式の運用が固まったら `carmel_line_channel_token` を設定し `carmel_line_mode=line` へ（フォールバック=メール）。
- **Cron**：WP-Cron が動作する前提（返済/車検/保険/会費/新着アラート/反響SLA/商談停滞/次アクション期日）。手動実行 `Carmel_Cron::instance()->run_daily()`。
- **未設定でも安全**：各連携は未設定なら no-op。段階的にキーを足して機能を有効化していけばよい。
