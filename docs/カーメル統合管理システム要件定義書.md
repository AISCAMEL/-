# 📘 カーメル統合管理システム 要件定義書

> 株式会社カーメル（carmelonline.jp）の中古車販売における「審査申込 〜 納車後アフターサポート」を一元管理する WordPress 統合システムの要件定義。
> マスタープロンプト v1.6 を実装可能な単位に整理したもの。

- **最終更新**：2026-06-08
- **対象バージョン**：システム仕様書 v1.6
- **開発体制**：自社開発（外注なし・AIサポートで実装）
- **インフラ**：WordPress on WPX（既存ドメイン `carmelonline.jp` に同一ドメインで追加）

## 4. 案件ステータスフロー

```
仮申込
  ↓
審査中（本部レビュー）
  ↓
審査OK / 審査NG
  ↓（OK時）
書類準備中
  ↓
契約完了（クラウドサイン：本部のみ）
  ↓
納車準備中（陸送手配・費用確定）
  ↓
納車済
  ↓
アフターサポート中
  ↓
案件クローズ
```

### 4.1 ステータス定義（実装用 slug 案）

| 表示名 | slug | フェーズ連動 |
|--------|------|-------------|
| 仮申込 | `provisional` | PHASE 1 |
| 審査中 | `screening` | PHASE 2 |
| 審査OK | `approved` | PHASE 3 |
| 審査NG | `rejected` | PHASE 3 |
| 書類準備中 | `doc_prep` | PHASE 4 |
| 契約完了 | `contracted` | PHASE 4 |
| 納車準備中 | `delivery_prep` | PHASE 5 |
| 納車済 | `delivered` | PHASE 6 |
| アフターサポート中 | `after_support` | PHASE 7 |
| 案件クローズ | `closed` | — |

### 4.2 ステータス変更時の自動処理

- WordPress → GAS webhook → **Asanaタスク自動生成**
- WordPress → プロライン → **ユーザーにLINE通知**
- WordPress → Gmail/メール → **関係者に通知**

## 5. 各ポータルの機能要件

### 5.1 ユーザーマイページ（/mypage）

案件ステータスに連動して UI が段階的に変化する **PHASE表示制御**。

| PHASE | 状態 | 表示内容 |
|-------|------|---------|
| 1 | 仮申込完了 | 申込内容確認・必要書類チェックリスト |
| 2 | 審査中 | 審査進捗バー・待機メッセージ |
| 3 | 審査結果 | OK：次のステップ案内 / NG：理由表示・再申込ボタン |
| 4 | 契約・書類 | 書類確認・署名待ち表示（クラウドサイン誘導） |
| 5 | 納車準備 | 納車日・陸送状況表示 |
| 6 | 納車済 | 保証書・各種書類ダウンロード・アフターサポート問い合わせ |
| 7 | アフターサポート | サポートチケット管理・FAQ・コミュニティ（bbPress）リンク |

### 5.2 加盟店ポータル（/store）

#### ダッシュボード
- 対応待ち案件数（ステータス別バッジ）
- 今週の納車予定（カレンダービュー）
- 未読通知・タスクアラート
- 月次売上サマリー

#### 案件管理
- カンバンビュー ＋ テーブルビュー切替
- 案件詳細：顧客情報・車両情報・書類・コメント
- ステータス変更 → 本部自動通知
- 陸送費自動計算（Google Maps API）

#### 書類管理

| 書類 | 操作 |
|------|------|
| 申込書 | GAS自動生成・PDF出力 |
| 審査結果通知 | 閲覧・印刷 |
| 売買契約書 | PDF生成（電子署名は加盟店独自対応） |
| 保証書 | PDF生成（価格表PDF方式） |
| 納車確認書 | PDF生成（顧客署名欄付） |
| 陸送依頼書 | 自動生成（陸送費計算連携） |

> ⚠️ クラウドサインは本部専用。加盟店はPDF発行のみ。

#### 在庫・車両管理
- Googleスプレッドシート連携表示
- 車両ステータス：販売中 / 商談中 / 売約済 / 納車済
- 陸送費計算（出発地〜納車先）

#### 顧客管理
- 担当顧客一覧・詳細
- 契約履歴・書類履歴・サポート履歴
- LINEメッセージ履歴（プロライン連携）

#### コミュニケーション
- 本部へのスレッドメッセージ（案件紐づけ）
- 顧客へのLINE配信（プロライン・テンプレート選択）
- 通知センター

#### タスク管理
- Asana連携（GAS経由・現行維持）
- 完了マーク → Asana自動同期

#### レポート
- Square決済データ集計（WooCommerce連携）
- 月次・週次・案件別売上
- CSV出力

#### 学習コンテンツ

## 7. 外部連携仕様

### 7.1 GAS（書類生成）
```
WordPress（wp_remote_post）
  → GAS WebアプリURL（doPost）
    → Googleドキュメントテンプレートをコピー
    → 差し込みデータで書類生成 → PDF変換
    → Google Driveに保存 → ダウンロードURLをWPに返却
    → Asanaタスク自動生成（案件ステータス連動）
```

### 7.2 Square決済
```
WooCommerce → Square for WooCommerce プラグイン
→ Square決済画面 → 決済完了
→ WooCommerce注文ステータス更新 → WP案件ステータスに反映
```

### 7.3 Google Maps API（陸送費計算）
```
出発地（加盟店住所）＋ 納車先（顧客住所）
→ Distance Matrix API → 距離（km）取得
→ 陸送費単価テーブル（本部設定）で自動計算
→ 案件詳細に自動表示・PDF生成時に自動挿入
```

### 7.4 プロライン（LINE通知）
```
案件ステータス変更 → WordPress → GAS or Webhook
→ プロライン API → ユーザーのLINEに通知
（テンプレート：審査結果・納車日確定・アフターサポート 等）
```

## 8. セキュリティ要件

- 各ページ（/mypage・/store・/hq）は**ログイン必須**
- 権限外のURLに直アクセス → **403 または /login にリダイレクト**
- 案件データは `customer_id` または `store_id` で**必ずフィルター**
  （他ユーザー・他加盟店のデータを絶対に見せない）
- GAS Webアプリは **Secret Token** で認証
- Square webhook は **Signature検証必須**
- WP標準フォームは **nonce必須**

## 10. 絶対に守る制約事項

1. 決済は **Squareのみ**（Stripeは使わない）
2. クラウドサインは **本部管理者のみ** 送付可能
3. 加盟店の電子署名は **加盟店独自の外部サービス**で対応（システム外）
4. 学習コンテンツは **Notion**で管理（WordPress内に実装しない）
5. Asanaは **現行GAS自動化を継続**（落ち着いたらWP移行）
6. **サブドメイン不使用**（carmelonline.jp に同一ドメインで追加）
7. 保証プランは固定価格ではなく **価格表PDF方式**
8. 在庫管理は **Googleスプレッドシート連携**（WP内DB管理しない）
