## 4. WordPress 再構築方針

SEO の観点およびユーザーの回遊性向上を目的とし、固定ページとカスタム投稿タイプを活用した構成へ再構築する。

- **固定ページ構成**：特徴・機能・料金・FAQ・フォーム等は固定ページとして分割管理。
- **カスタム投稿タイプ `cases` の採用**：導入事例は拡張・更新を見越しカスタム投稿タイプで管理。
- **静的フロントページ設定**：「設定 > 表示設定」でトップページを固定ページに割り当て。
- **推奨テーマ構成**：Elementor Pro + Hello Elementor を第一候補。LP 型デザインの再現性が高く、ノーコードでのレイアウト構築・動的コンテンツ連携に最適。
- **代替案（要確認）**：Elementor Pro が使用できない場合、「SWELL」や「Astra + Elementor（無料版）」も検討可能。

> **【本リポジトリでの実装方針／修正点】** Elementor Pro のライセンスが「要確認」ステータスのため、**ライセンス非依存の自作テーマ `apprex`（Hello Elementor 思想を踏襲した独立テーマ）** として実装。cases・FAQ は CPT＋ACF で wp-admin から編集可能とし、セクション 9「クライアント自身が編集可能」の要件を満たす。後から Elementor を導入しても共存できる構造とする。

## 8. カスタム投稿タイプ設計（cases）

ACF の利用を推奨し、以下のフィールドを定義する。

- 業種（テキストまたはタクソノミー）
- 成果指標1（例：売上+150%）／成果指標2（例：会員数10,000人）
- 開発期間（例：2週間）
- 利用機能（チェックボックスまたはテキスト）

**役割**：一覧（アーカイブ）で業種ごとに絞り込み可能なグリッドを表示し、詳細（シングル）で課題解決プロセスを記述する。

## 9. WordPress 実装ルール

- **命名規則**：クラス名やスラッグは英語小文字のケバブケース（例 `free-trial`）で統一。
- **画像最適化**：全画像を WebP に変換し、遅延読み込み（Lazy Load）を適用。
- **レスポンシブ**：モバイルファースト設計。タップ領域・スタック順序を最適化。
- **CTA導線**：各下層ページ末尾に共通 Final CTA セクションを必ず配置。
- **動的設計**：FAQ や導入事例はクライアントが後から容易に追加・修正できるようにする。

## 11. 制作用 AI プロンプト（原文保存）

> あなたはプロフェッショナルな WordPress エンジニア・Web デザイナーです。提供された実装指示書に基づき、APPREX のトップページを「Elementor Pro」と「Hello Elementor」を使用して構築するための設計・CSS 案を出力してください。
>
> 1. **デザイン方針**：ビジネス向けの信頼感とモダンさを兼ね備えたクリーンなデザイン。余白を十分に取り、モバイルファーストで構築。
> 2. **動的コンテンツ**：導入事例はカスタム投稿タイプ「cases」と ACF を使用し、Loop Builder でカード形式に出力。
> 3. **主要ウィジェット**：CTA ボタン（無料体験・相談）の配置／FAQ は Accordion／料金表（トライアル/スタート/ビジネス）は Price Table で横並び比較。
> 4. 表示速度に配慮し、不要な DOM のネストを避ける。
> 5. まずは Hero セクションおよび Stats セクションの Flexbox Container を用いた具体的なレイアウト構造案を提示。
