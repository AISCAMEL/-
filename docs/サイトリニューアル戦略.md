## 1. リニューアル戦略サマリー

### 課題（現状）
現サイト（https://aiscompany.jp/）は世界観・理念は伝わるが、初見の法人顧客に対して
**「何の会社か／誰向けか／何を頼めるか／なぜアイズか」** が瞬時に伝わらない。
結果として、問い合わせという行動につながりにくい。

### 今回の方針
「理念を語るサイト」から **「相談につながる営業型コーポレートサイト」** へ転換する。

| 観点 | Before（現状の課題） | After（リニューアル方針） |
| --- | --- | --- |
| 第一印象 | 抽象的で事業が不明瞭 | ヒーローで「自動車業界のDX・販売支援」を一瞬で明示 |
| サービス | 20以上の業務が羅列 | **3本柱**に集約し、詳細は下層で分解 |
| 訴求順序 | 会社理念が先行 | 顧客の**課題 → 解決策 → 選ばれる理由 → 流れ**の順 |
| CV導線 | 弱い | 全ページ末にCTAバナー、ヘッダー常設「無料相談」 |
| 拡張性 | 静的・追加困難 | 実績・記事・サービスをデータ駆動で追加可能 |

### 成功指標（KPI 例）
- 主要KPI：問い合わせ数／問い合わせCVR
- 補助KPI：サービス詳細ページ閲覧率、FAQ閲覧→Contact遷移率、直帰率、平均滞在時間

## 2. ブランドポジション整理

- **ブランド定義**：AIS = Always Innovation Solutions（常に、新しい解決策を）
- **一言で**：自動車業界に強い、DX・販売・集客の「戦略から実行まで」伴走パートナー
- **独自性（差別化の核）**：**「自動車業界の知見 × IT/DXの実装力」** の掛け算。
  さらに **創業支援〜開発〜集客まで** をフェーズ横断で一社完結できる。

### ブランドトーン
未来志向／先進的／誠実／専門的／親しみはあるが軽すぎない／わかりやすい。

### ポジショニング（同時に成立させる4象限）
```
        専門的・上流（戦略）
              ▲
              │      ● アイズ
  IT/Web寄り ─┼───────────────→ 自動車業界寄り
              │   （戦略から実行まで／相談しやすい）
              ▼
        実務・実行（手を動かす）
```
- 「戦略だけのコンサル」でも「制作だけの下請け」でもなく、**上流から実行まで**をカバー。

## 9. デザインシステム方針

- **カラー**：
  - 背景ダーク：`ink-900 #0b1120`／`ink-800 #0f172a`
  - ブランドブルー：`brand-600 #2563eb`（CTA・主要アクション）
  - アクセント・シアン：`accent-500 #06b6d4` / `accent-400 #22d3ee`（先進性の差し色）
  - テキスト：`ink-800`（本文）／`ink-600`（補足）
- **タイポ**：Noto Sans JP（400/500/700）。見出し`tracking-tight` + `text-balance`、本文`leading-relaxed` + `text-pretty`。
- **余白**：セクション縦`py-16 / sm:py-24`、コンテナ最大幅`1200px`。
- **ボタン**：`rounded-full`、primary=ブランドブルー、secondary=白＋枠線、ghost=ダーク背景上。
- **カード**：`rounded-2xl` + `shadow-card`、ホバーで浮上（`-translate-y-1`）。
- **アイコン**：軽量インラインSVG（`Icon.tsx`）。stroke統一でトーンを揃える。
- **アクセシビリティ**：`focus-visible`リング、スキップリンク、`aria-*`、コントラスト確保。
- **アニメーション**：ホバーの微小トランジションのみ。過剰演出はしない。

## 10. 実装アーキテクチャ

- **Framework**：Next.js 14（App Router）/ React 18 / TypeScript（strict）
- **Styling**：Tailwind CSS v3（`tailwind.config.ts`にデザイントークン集約）
- **構成思想**：**コンテンツ（`src/content/*`）とUI（`src/components/*`）を分離**。
  文言・サービス・FAQ・実績・記事はすべてデータ化 → CMS移行が容易。
- **レンダリング**：全ページ静的生成（SSG）。動的ルートは`generateStaticParams`。
- **SEO**：`metadata`API、`sitemap.ts`、`robots.ts`、JSON-LD（Organization / FAQPage）。
- **パフォーマンス**：画像は`next/image`前提、JS最小（First Load JS 約87–95kB）、フォント`display:swap`。

```
src/
├─ app/                 ルーティング（各ページ + sitemap/robots/not-found）
├─ components/
│   ├─ layout/          Header, Footer, PageHero, CtaBanner
│   ├─ ui/              Button, Container, Section, Accordion, Icon
│   ├─ home/            トップ各セクション
│   └─ contact/         ContactForm
└─ content/             site, navigation, services, home, faq, works, news, company
```

## 12. 今後の拡張方針（CMS連携・事例追加）

1. **当面（コードベース運用）**：`src/content/*.ts` の配列に追記するだけで、実績・記事・サービスが増える設計。`isPlaceholder: false` でサンプル表記が消える。
2. **CMS連携（推奨：headless CMS）**：microCMS / Contentful / Sanity などを採用。
   - `content/*.ts` の型（`ServiceDetail` / `Work` / `NewsItem`）をAPIレスポンス型として再利用。
   - 各ページのデータ取得を `fetch`（ISR：`revalidate`）へ置換するだけでUIは無改修。
3. **記事のリッチ化**：News本文を文字列配列からMDX / リッチテキストへ。
4. **フォーム**：`ContactForm` を API Route もしくは外部フォーム（formrun / HubSpot 等）へ接続。スパム対策（reCAPTCHA / honeypot）を追加。
5. **計測**：GA4 / Search Console / ヒートマップを導入し、CVRを継続改善。

### 差し替えが必要な placeholder 一覧
- 会社概要（代表者・設立・所在地）= `src/content/company.ts`
- 連絡先（メール・電話・住所）= `src/content/site.ts`
- 実績・事例（`works.ts`）、お知らせ（`news.ts`）の実データ
- プライバシーポリシー本文（法令・運用に合わせて確定）
- 本番ドメイン（`site.url`）とOGP画像
