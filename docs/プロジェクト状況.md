### 仕組み（全体）
- **ジャンル一元管理**：`assets/js/genres.js`（2階層 GROUPS/LIST・全`live`）を編集→LPカード・ジャンルハブ・全フッター・専用LP・掛け合わせLPに自動反映。`tools/gen-genre.js`/`gen-area.js` が静的生成、`tools/_cross.js` が掛け合わせ対象を一元管理。
- **フォーム送信**：`assets/js/buymo.js` の `ENDPOINT` に GAS `/exec`→査定/問合せ/パートナー応募が `type:"buymo"` で記録＋通知＋ステップメール登録＋案件自動生成。来訪元の `genre/pref/est` を引き継ぎ、送信成功で `buymo-thanks.html` へ遷移。空欄はデモ。
- **査定シミュレーター**：`simulator-buy.js`。トップLPに加え**全ジャンルLP・掛け合わせLP**へ展開（`[data-buymo-sim]`自動初期化・車種/状態プリセット・概算CTAに `genre/pref/est` 付与）。パーツ系ジャンルは非表示。
- **QA自動化**：`tools/check-links.js`（リンク健全性）＋`tools/seo-check.js`（JSON-LD/SEO要素/データ整合）。`.github/workflows/qa.yml` でpush毎に実行。
- **ステップメール**：`gas/StepMail.gs`（Day0/1/3/7・会員ページ誘導・配信停止）。`runStepMails` をトリガー登録。WPからは `type:"stepmail"` で発火。
- **業務システム**：`hq-common.js` が案件データ（`buymo_cases`）を共有。看板（`board.js`）・リード（`hq-leads.js`）・加盟店（`hq-stores.js`）・レポート（`report.js`）・会員マイページ（`member.js`）が同一データで連動。GAS：`Board.gs`（`type:"case"/"note"`・doGet `cases`/`mycase`）。
- **認証**：`auth.js`＋`gas/Auth.gs`（ソルト付きSHA-256・セッショントークン）。業務ページは `AUTH.guard()`。書き込みはトークン必須。詳細・限界は `docs/BUYMO_認証設計.md`。
- **アカデミー**：`academy.js`（コース/進捗/クイズ/修了証）。
- **チャットボット**：`chatbot.js`（ユーザー/加盟店 切替・ルールベース）。
- **お知らせ**：`notices.js`（加盟店バナー）＋ `hq-notices.html`。
- **計測**：`analytics.js`（GA4・`generate_lead`/`simulate`）。
- **ページ生成**：`tools/gen-area.js` / `tools/gen-genre.js` / 共通 `tools/_layout.js`。
- **技術SEO**：`sitemap.xml`・`robots.txt`・canonical・JSON-LD・OGP。構造図 `docs/buymo_seo_mindmap.*`。

> 詳細な一覧（全ページ・モジュール・GASエンドポイント・設定箇所・localStorageキー・公開チェックリスト）は **`docs/BUYMO_全体マップ.md`**。

## 4. ローカルでプレビューする
静的サイトなのでサーバー不要。簡易サーバー例：
```bash
cd site
python3 -m http.server 8000
# ブラウザで http://localhost:8000/index.html を開く
```
会員サイトは index → 無料会員登録 → mypage の順で確認（データはブラウザ内に保存）。
