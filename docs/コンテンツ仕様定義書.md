# 合同会社アイズ コーポレートサイト コンテンツ仕様定義書（WordPress版）

> 目的：実際のコピー文を書き始める前に、**「各ページに何を・どんな構造で・どの項目を載せるか」** を確定させる。
> 本書が「中身の設計図」となり、以降はこの枠を文章で埋めていく。
> `（要記入）`＝お客様にご提供・ご確認いただく項目。`（こちらで作成）`＝コピーを当方で起案。

---

## 0. 前提・全体方針

| 項目 | 定義 |
| --- | --- |
| プラットフォーム | WordPress（現行サイトを継続利用） |
| 目的 | 「何の会社か／誰向けか／何を頼めるか／なぜアイズか」を即伝え、**問い合わせにつなげる** |
| サービス見せ方 | 多数の業務を **3本柱** に集約して提示 |
| 主軸 | 自動車業界支援を主軸に、創業支援・Web/開発支援を整理 |
| トーン | 未来志向・誠実・専門的・わかりやすい（BtoB） |
| 主要CV | お問い合わせ／無料相談 |

## 5. 確認したい論点（仕様を固めるための質問）

1. **サービスの3本柱の分け方**：この3分類（自動車／創業／Web開発）で確定でよいか？
2. **主軸の強調度**：自動車業界支援をどのくらい前面に出すか（トップの占有比率）。
3. **実績の有無**：現時点で公開できる実績はあるか／サンプル運用で開始するか。
4. **ターゲットの優先順位**：最も獲りたい問い合わせは「自動車新規参入」か「DX」か「Web/開発」か。
5. **フォームのプラグイン指定**：既存で使っているものがあるか。
6. **トーン**：現状コピー案（誠実・専門的・わかりやすい）で問題ないか。
