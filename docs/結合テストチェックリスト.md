# 結合テスト チェックリスト（本番リリース前）

対象：`carmel-core`（要件定義 v1.6）／実 WordPress 環境での動的検証用。
使い方：上から順に実施し、各項目の「期待結果」を満たすか確認してチェック。

---

## 0. 事前準備

- [ ] WordPress に `carmel-core` を配置し**有効化**（エラーなく有効化できる）
- [ ] 有効化で CPT 9種・ロール4種が登録される（管理画面メニュー＋ユーザー権限で確認）
- [ ] 必要プラグイン導入：ACF Pro／WooCommerce＋Square for WooCommerce／（任意）CF7 or Gravity Forms／bbPress／WooCommerce Subscriptions
- [ ] 固定ページ作成＋ショートコード設置：
  - [ ] `/login`（ログインフォーム）
  - [ ] `/mypage` → `[carmel_mypage]` ＋ `[carmel_upload]`
  - [ ] `/store` → `[carmel_store]`
  - [ ] `/hq` → `[carmel_hq_screening]` `[carmel_hq_board]` `[carmel_hq_contracts]` `[carmel_hq_reports]`
  - [ ] 学習ページ → `[carmel_learning]`
- [ ] 連携キーを設定（プロライン/LINE WORKS/Slack/GAS/Maps/Square/MF契約/Notion/community）
- [ ] 外部 webhook 登録：Square `/wp-json/carmel/v1/square-webhook`、GAS `/wp-json/carmel/v1/gas-callback`、MF契約 `/wp-json/carmel/v1/mf-contract-callback`
- [ ] テストユーザー作成：hq_admin / store_owner（store_id付）/ store_staff（store_id付）/ customer
- [ ] テスト用 `carmel_store` を1件作成（store_address・notion_url・会費フィールド設定）

## 1. アクセス制御（権限）

- [ ] 未ログインで `/mypage` `/store` `/hq` → `/login` にリダイレクト
- [ ] customer で `/store`・`/hq` → 403
- [ ] store_staff で `/hq` → 403、`/store` は表示
- [ ] hq_admin で全ポータル表示
- [ ] 他店の store_owner が他店案件を操作 → 403（行レベル制御）

## 2. 申込 → アカウント自動発行

- [ ] 申込フォーム送信（または REST `POST /wp-json/carmel/v1/application` ＋トークン）
- [ ] `carmel_deal` が作成され、`deal_status=provisional`（loan）/`appraisal_request`（buyback）/`lease_request`（lease）
- [ ] 顧客ユーザーが `customer` ロールで自動発行（既存メールは再利用）
- [ ] 「申込受付＋アカウント案内」通知が顧客へ届く（LINE未連携時はメールにフォールバック）
- [ ] パスワード設定リンクからログインできる

## 3. AIスコアリング（GAS・loan）

- [ ] loan 申込の数分後、`ai_score`/`score_rank` が保存され `deal_status=scored` に前進
- [ ] `/hq` 審査画面の「AIスコア」列にスコア表示
- [ ] GAS 未設定時：エラーで全体停止せず、Slack に失敗通知（`system_error`）

## 4. 審査（本部 `[carmel_hq_screening]`）

- [ ] 審査待ち一覧に対象案件が出る
- [ ] 「審査開始」→ `screening`
- [ ] 「審査OK」→ `approved`、顧客へ審査結果通知、`/mypage` がPHASE③で通過表示
- [ ] 「審査NG（理由入力）」→ `rejected`、顧客通知にNG理由反映、`/mypage` に理由＋再申込導線
- [ ] store_staff には審査操作が出ない（cap）

## 5. アサイン → 加盟店工程（`[carmel_store]`）

- [ ] 本部が案件に `store_id` を設定（matched）→ 加盟店へアサイン通知
- [ ] 加盟店ダッシュボードに自店案件が出る（他店は出ない）
- [ ] 「書類準備へ」で `doc_prep`、`doc_prep` では「本部の手続き待ち」表示
- [ ] 納車準備へ進める際に納車日入力 → `delivery_prep`、顧客へ納車日通知
- [ ] `delivery_prep` 進入で陸送費が自動計算され `/mypage` に表示（Maps設定時）

## 6. 契約（マネーフォワード契約 `[carmel_hq_contracts]`・本部のみ）

- [ ] 本部で「マネーフォワード契約 送付」→ `mf_contract_status=sent`、顧客へ署名案内、契約書 `carmel_document` 作成
- [ ] store_staff/owner には契約画面が出ない（cap）
- [ ] 署名完了 webhook 受信 → `contracted` に自動前進、署名済PDF保存
- [ ] 署名 webhook のトークン不一致は拒否される

## 7. 納車・在庫連動

- [ ] 案件に `vehicle_id` を紐付け、`matched`→車両「商談中」/`contracted`→「売約済」/`delivered`→「納車済」
- [ ] `delivered` で顧客へ通知、`/mypage` がPHASE⑥

## 8. 決済（Square / WooCommerce）

- [ ] 注文に `carmel_deal_id`/`carmel_payment_type` を付与し完了 → `payment_{type}_status=paid`、顧客へ完了通知
- [ ] Square webhook（payment.updated/COMPLETED）→ 入金記録（JPYは円単位で正しい金額）
- [ ] 署名不一致の Square webhook は 401/拒否
- [ ] 注文失敗 → `payment_failed` 通知（店舗/Slack）

## 9. 定期処理（WP-Cron）

- [ ] `Carmel_Cron::instance()->run_daily()` を手動実行して検証
  - [ ] 返済期日3日前/前日/当日でリマインダー（未払いのみ）
  - [ ] 延滞で `delay_days`/`delay_interest` 書き戻し、1/5/14日目に督促
  - [ ] 車検90/60/30日前・保険90/30日前で通知
  - [ ] 会費請求7日前/前日で案内、期日超過で `expired`＋通知
- [ ] 同日再実行で**重複通知が出ない**（冪等性）
- [ ] `run_weekly()` で週次レポートが Slack＋本部メールへ

## 10. 通知4チャネル

- [ ] プロライン（顧客LINE）送信
- [ ] LINE WORKS（社内）送信
- [ ] Slack（システム/運用）送信
- [ ] メール（正式・フォールバック）送信
- [ ] LINE失敗時にメールへ自動フォールバックする
- [ ] `carmel_notify_log` に全送信が記録される

## 11. 書類アップロード（`[carmel_upload]`）

- [ ] 顧客が自分の案件にJPG/PNG/PDFをアップロードできる
- [ ] 10MB超・非対応形式は拒否
- [ ] 保存先 `uploads/carmel-secure/...` に**直URLアクセスすると拒否**（Apache）/ Nginx は location 設定確認
- [ ] ダウンロードリンク（nonce付）で本人・担当店・本部は閲覧可、無関係ユーザーは403

## 12. レポート（`[carmel_hq_reports]`）

- [ ] KPI（総数・成約数・転換率・売上合計）が正しい
- [ ] 売上内訳・ステータス分布・加盟店別が表示
- [ ] store_owner は自店のみ集計
- [ ] CSVダウンロード（Excelで文字化けしない＝BOM）

## 13. カンバン（`[carmel_hq_board]`・本部）

- [ ] 種別タブ・加盟店フィルターが効く
- [ ] カードをD&Dで列移動 → ステータス変更が保存され、通知・在庫・監査が連動
- [ ] 失敗時はカードが戻りエラー表示
- [ ] 非HQでは表示されない

## 14. 会費・コミュニティ

- [ ] WCサブスク有効化で店舗 `membership_status=active`＋次回請求日
- [ ] サブスク失効/解約で `expired`＋通知
- [ ] 手動（ACF会費フィールド）でも状態管理できる
- [ ] `[carmel_learning]`：店舗/本部に学習（Notion）リンク、全員にコミュニティ（bbPress）リンク

## 15. 監査・セキュリティ最終

- [ ] ステータス変更が `_carmel_status_history`（誰が/いつ/from→to）に記録
- [ ] 各フォーム/操作で nonce 必須（改ざんPOSTは400/403）
- [ ] REST 申込口はトークン未設定時に拒否
- [ ] HTTPS で全画面・webhook が動作
- [ ] （Nginx）`uploads/carmel-secure/` の直アクセス拒否設定済み
