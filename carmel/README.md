# carmel — 在庫STEP UI 修正メモ

対象: **carmelonline.jp**（WordPress / カスタム投稿タイプ `portfolio`＝在庫）の
在庫編集画面に表示される「**車両入力 STEP UI**」（メタボックス `carmel_step_ui`）。

> ⚠️ このSTEP UI 本体のコードは本リポジトリ管理外（WordPress 上の
> WPCode スニペット／メタボックス出力PHP）です。ここには**追加で当てる修正**を置いています。

---

## 修正1: ステップ2の装備が「ちゃんとチェックされない」問題

### 症状
ステップ1→2に進んで装備（ナビ・安全装備・快適装備など）を選んでも、
通常の「更新」では保存されず、チェックが反映されない。

### 原因
ステップ2の装備チェックボックス `.cs-equip-check` は **表示専用の独立要素**で、
実際に保存される ACF の装備フィールド（基本装備／オーディオ／カーナビ・TV／
シート・内装／ドア・外装／安全装備）と**連動していない**。
ステップ2の値は STEP6「ACFに一括反映して保存」AJAX（`acf_equip`）でしか渡らず、
通常保存では捨てられていた。

### 対応
`step-ui-equip-bridge.js` を追加。ステップ2で装備を選んだ瞬間に、
対応する ACF チェックボックスをオンにする（＝通常の「更新」でも確実に保存）。
既存レコードを開いたときは ACF の状態をステップ2へ初期反映する。

### 導入手順（WPCode）
1. 管理画面 → **コードスニペット → ＋スニペットを追加**
2. コードタイプ: **JavaScript Snippet**
3. `step-ui-equip-bridge.js` の中身を貼り付け
4. 挿入位置: **管理画面のみ（Admin area only）**
5. 必要なら対象を在庫編集画面に限定（`post_type=portfolio`）
6. 有効化して保存

### 注意（マッピングについて）
ステップ2の装備名と ACF の装備名は語彙が一致しない項目があるため、
**ACF 側に対応項目があるものだけ**連動させています。
ACF に該当が無い項目（Apple CarPlay / Android Auto / レーンアシスト /
ドライブレコーダー / ベンチレーションシート / 革シート / パノラマルーフ /
プッシュスタート / LEDヘッドライト / フォグランプ / 純正ナビ・社外ナビ /
車歴系 など）は連動対象外です。必要なら ACF にフィールドを追加し、
`EQUIP_MAP` に対応を追記してください。

## 修正2: STEP UI ↔ ACF ブリッジ統合（`step-ui-acf-bridge.php`）

「在庫ページの自動投稿（＝STEP1完了時の基本情報→ACF自動入力）を綺麗に直したい」への対応。

### これまでの構成（バラバラ）
- `carmel_step1_autofill`（PHP）… 基本情報＋タイトルを ACF へ自動入力
- `step-ui-equip-bridge.js`（JS）… STEP2 装備を ACF チェックへ連動

ヘルパー（`setAcf` 等）もトリガーも二重管理で、取りこぼし・初期反映抜けがあった。

### 統合後（1本）
`step-ui-acf-bridge.php` に **基本情報・タイトル・装備・初期反映** をまとめ、
同じヘルパー／同じトリガーで動かす。旧2スニペットはこれで置き換え。

### 主な改善点
1. **2スニペットを1本に統合** — 保守箇所が1つに。
2. **空値の同期** — STEP1 で値を消すと ACF 側も空に揃う（旧版は古い値が残った）。
3. **保存直前に最終同期**（`#post` の submit）— STEP2以降で直した値の取りこぼし防止。
4. **既存レコードの初期反映** — 装備チェックを ACF から STEP UI へ戻す。
5. ステップ移動（次へ/戻る/ナビ）でも全同期。

### 導入手順（WPCode）
1. コードタイプ **PHP Snippet**、挿入位置 **自動挿入・どこでも(Run Everywhere)**
2. `step-ui-acf-bridge.php` の `<?php` 直後〜末尾までを貼り付け
3. 有効化
4. **旧スニペット2本（`carmel_step1_autofill` / 装備JS）は無効化**

> ※ `step-ui-equip-bridge.js` は統合版に内包済み。単体運用を続ける場合のみ残す。
